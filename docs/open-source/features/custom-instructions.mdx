自定义指令可让您准确决定 Mem0 从对话中记录哪些事实。定义一个重点提示，给出一些示例，Mem0 将仅添加与您的用例匹配的内存。

<信息>
  **您将在以下情况下使用它...**
  - 项目需要特定领域的事实（订单号、客户信息），而不存储随意的闲聊。
  - 您已经有一个清晰的记忆模式，并希望法学硕士遵循它。
  - 您必须防止不相关的详细信息进入长期存储。
</信息>

<警告>
  过于宽泛的提示会导致忽略不相关的事实。严格遵守指示并用真实的成绩单进行测试。
</警告>

<注意>
  “custom_fact_extraction_prompt”参数已重命名为“custom_instructions”。如果您要从旧版本升级，请相应地更新您的配置。
</注>

---

## 特征剖析- **提示说明：** 描述要保留哪些实体或短语。特定的指导使提取器保持专注。
- **少量示例：** 显示正面和负面案例，以便模型复制正确的格式。
- **结构化输出：** 响应返回带有“facts”数组的 JSON，Mem0 将其转换为单独的内存。
- **LLM 配置：** `custom_instructions` (Python) 或 `customInstructions` (TypeScript) 与您的模型设置一起存在。

<手风琴组>
  <Accordion title="提示蓝图">
    1. 说明允许的事实类型。
    2. 包括反映生产消息的简短示例。
    3. 显示空 (`[]`) 和已填充的输出。
    4. 提醒模型仅返回带有“facts”键的 JSON。
  </手风琴>

## 配置它

### 编写自定义提示

<代码组>```python Python
custom_instructions = """
Please only extract entities containing customer support information, order details, and user information.
Here are some few shot examples:

Input: Hi.
Output: {"facts" : []}

Input: The weather is nice today.
Output: {"facts" : []}

Input: My order #12345 hasn't arrived yet.
Output: {"facts" : ["Order #12345 not received"]}

Input: I'm John Doe, and I'd like to return the shoes I bought last week.
Output: {"facts" : ["Customer name: John Doe", "Wants to return shoes", "Purchase made last week"]}

Input: I ordered a red shirt, size medium, but received a blue one instead.
Output: {"facts" : ["Ordered red shirt, size medium", "Received blue shirt instead"]}

Return the facts and customer information in a json format as shown above.
"""
```

```ts TypeScript
const customInstructions = `
Please only extract entities containing customer support information, order details, and user information.
Here are some few shot examples:

Input: Hi.
Output: {"facts" : []}

Input: The weather is nice today.
Output: {"facts" : []}

Input: My order #12345 hasn't arrived yet.
Output: {"facts" : ["Order #12345 not received"]}

Input: I am John Doe, and I would like to return the shoes I bought last week.
Output: {"facts" : ["Customer name: John Doe", "Wants to return shoes", "Purchase made last week"]}

Input: I ordered a red shirt, size medium, but received a blue one instead.
Output: {"facts" : ["Ordered red shirt, size medium", "Received blue shirt instead"]}

Return the facts and customer information in a json format as shown above.
`;
```
</代码组>

<提示>
  保持示例对简短并反映您在真实用户消息中看到的大小写、标点符号和语气。
</提示>

### 在配置中加载提示

<代码组>```python Python
from mem0 import Memory

config = {
    "llm": {
        "provider": "openai",
        "config": {
            "model": "gpt-5-mini",
            "temperature": 0.2,
            "max_tokens": 2000,
        }
    },
    "custom_instructions": custom_instructions,
}

m = Memory.from_config(config)
```

```ts TypeScript
import { Memory } from "mem0ai/oss";

const config = {
  llm: {
    provider: "openai",
    config: {
      apiKey: process.env.OPENAI_API_KEY ?? "",
      model: "gpt-4-turbo-preview",
      temperature: 0.2,
      maxTokens: 1500,
    },
  },
  customInstructions: customInstructions,
};

const memory = new Memory(config);
```
</代码组>

<信息图标=“检查”>
  初始化后，使用已知示例运行快速“add”调用，并确认响应拆分为单独的事实。
</信息>

## 查看实际效果

### 示例：订单支持内存

<代码组>```python Python
m.add("Yesterday, I ordered a laptop, the order id is 12345", user_id="alice")
```

```ts TypeScript
await memory.add("Yesterday, I ordered a laptop, the order id is 12345", { userId: "user123" });
```

```json Output
{
  "results": [
    {"memory": "Ordered a laptop", "event": "ADD"},
    {"memory": "Order ID: 12345", "event": "ADD"},
    {"memory": "Order placed yesterday", "event": "ADD"}
  ],
  "relations": []
}
```
</代码组>

<信息图标=“检查”>
  输出仅包含提示中描述的事实，每个事实都存储为单独的内存条目。
</信息>

### 示例：过滤掉不相关的消息

<代码组>```python Python
m.add("I like going to hikes", user_id="alice")
```

```ts TypeScript
await memory.add("I like going to hikes", { userId: "user123" });
```

```json Output
{
  "results": [],
  "relations": []
}
```
</代码组>

<提示>
  空“结果”显示提示已成功忽略目标域之外的内容。
</提示>

## 验证该功能是否正常工作

- 记录部署期间的每个调用并确认“facts”数组与您的架构匹配。
- 检查不相关的消息是否返回空的“结果”数组。
- 每当您编辑提示时运行回归示例，以确保先前接受的事实仍然通过。

---

## 最佳实践

1. **精确：** 准确指出您想要捕获的类别或字段。
2. **显示负面案例：** 包含应生成“[]”的示例，以便模型学会跳过它们。
3. **保持 JSON 严格：** 避免额外的键；只返回“facts”以简化下游解析。
4. **版本提示：** 使用版本号跟踪提示更改，以便您可以快速回滚。
5. **定期检查输出：** 抽查存储的记忆以尽早发现偏差。
