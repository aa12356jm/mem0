Mem0 1.0.0 中增强的元数据过滤功能使您可以跨内存元数据运行复杂的查询。将比较、逻辑运算符和通配符匹配组合为零，以获取代理所需的确切内存。

<信息>
  **你会在以下情况下使用它......**
  - 在返回上下文之前，检索必须尊重多个元数据条件。
  - 您需要在单个查询中混合数字、布尔和字符串过滤器。
  - 代理依赖于确定性过滤而不是单独的广泛语义搜索。
</信息>

<警告>
  增强过滤需要 Mem0 1.0.0 或更高版本以及支持您启用的运算符的矢量存储。不受支持的运算符会退回到简单的相等过滤器。
</警告>

<注意>
  TypeScript SDK 接受此处所示的相同过滤器形状 - 将字典转置为对象并按原样重复使用键。
</注>

    | `gt` / `gte` |大于/大于或等于 |按分数、置信度或任何数字字段对结果进行排名。 |

### 元数据选择器

当您需要对元数据字段进行直接匹配时，请从键值过滤器开始。```python
from mem0 import Memory

m = Memory()

# Search with simple metadata filters
results = m.search(
    "What are my preferences?",
    filters={"user_id": "alice", "category": "preferences"}
)
```
<信息图标=“检查”>
  预计只有标有“category=”preferences“”的内存才会返回给定的“user_id”。
</信息>

### 比较运算符

分层大于/小于比较，按分数、置信度或任何数字字段对结果进行排名。相等助手（`eq`、`ne`）使字符串和布尔检查保持明确。```python
# Greater than / Less than
results = m.search(
    "recent activities",
    filters={
        "user_id": "alice",
        "score": {"gt": 0.8},
        "priority": {"gte": 5},
        "confidence": {"lt": 0.9},
        "rating": {"lte": 3}
    }
)

# Equality operators
results = m.search(
    "specific content",
    filters={
        "user_id": "alice",
        "status": {"eq": "active"},
        "archived": {"ne": True}
    }
)
```
### 基于列表的运算符

当您想要预先批准或排除特定值而不编写多个相等检查时，请使用“in”和“nin”。```python
# In / Not in operators
results = m.search(
    "multi-category search",
    filters={
        "user_id": "alice",
        "category": {"in": ["food", "travel", "entertainment"]},
        "status": {"nin": ["deleted", "archived"]}
    }
)
```
<信息图标=“检查”>
  验证响应仅包含白名单类别中的记忆，并忽略任何具有已存档或已删除状态的记忆。
</信息>

### 字符串运算符

`contains` 和 `icontains` 捕获子字符串匹配，可以轻松扫描关键字的描述或标签，而无需检索不相关的记忆。```python
# Text matching operators
results = m.search(
    "content search",
    filters={
        "user_id": "alice",
        "title": {"contains": "meeting"},
        "description": {"icontains": "important"},
        "tags": {"contains": "urgent"}
    }
)
```
### 通配符匹配

允许字段具有任何值，同时仍然要求字段存在——当字段的存在很重要时，这很方便。```python
# Match any value for a field
results = m.search(
    "all with category",
    filters={
        "user_id": "alice",
        "category": "*"
    }
)
```
### 逻辑组合

将过滤器与“AND”、“OR”和“NOT”组合起来以表达复杂的决策树。嵌套逻辑运算符来编码多分支工作流程。```python
# Logical AND
results = m.search(
    "complex query",
    filters={
        "AND": [
            {"user_id": "alice"},
            {"category": "work"},
            {"priority": {"gte": 7}},
            {"status": {"ne": "completed"}}
        ]
    }
)

# Logical OR
results = m.search(
    "flexible query",
    filters={
        "AND": [
            {"user_id": "alice"},
            {
                "OR": [
                    {"category": "urgent"},
                    {"priority": {"gte": 9}},
                    {"deadline": {"contains": "today"}}
                ]
            }
        ]
    }
)

# Logical NOT
results = m.search(
    "exclusion query",
    filters={
        "AND": [
            {"user_id": "alice"},
            {
                "NOT": [
                    {"category": "archived"},
                    {"status": "deleted"}
                ]
            }
        ]
    }
)

# Complex nested logic
results = m.search(
    "advanced query",
    filters={
        "AND": [
            {"user_id": "alice"},
            {
                "OR": [
                    {"category": "work"},
                    {"category": "personal"}
                ]
            },
            {"priority": {"gte": 5}},
            {
                "NOT": [
                    {"status": "archived"}
                ]
            }
        ]
    }
)
```
<信息图标=“检查”>
  检查响应元数据 - 每个返回的内存都应该完全满足组合逻辑树。如果结果看起来太宽泛，请记录发送到矢量存储的原始过滤器。
</信息>

## 配置它

调整您的矢量存储，以便过滤繁重的查询保持快速。对您经常过滤的字段进行索引，并在评估顺序中保留复杂的检查以供稍后使用。```python
# Ensure your vector store supports indexing on filtered fields
config = {
    "vector_store": {
        "provider": "qdrant",
        "config": {
            "host": "localhost",
            "port": 6333,
            "indexed_fields": ["category", "priority", "status", "user_id"]
        }
    }
}
```
<信息图标=“检查”>
  启用索引后，对相同的查询进行基准测试 - 一旦存储可以在矢量评分之前修剪索引字段上的文档，延迟就会下降。
</信息>

<提示>
  在范围或文本条件之前将简单的 key=value 过滤器放在索引字段上，以便商店尽早修剪结果。
</提示>```python
# More efficient: Filter on indexed fields first
good_filters = {
    "AND": [
        {"user_id": "alice"},
        {"category": "work"},
        {"content": {"contains": "meeting"}}
    ]
}

# Less efficient: Complex operations first
avoid_filters = {
    "AND": [
        {"description": {"icontains": "complex text search"}},
        {"user_id": "alice"}
    ]
}
```
<信息图标=“检查”>
  当您重新排序过滤器以使索引字段排在第一位（“good_filters”示例）时，查询通常比“avoid_filters”模式返回得更快，在“avoid_filters”模式中，昂贵的文本搜索先于简单检查运行。
</信息>

矢量存储支持各不相同。发货前确认运营商承保范围：

</警告>

### 从早期过滤器迁移```python
# Before (v0.x) - simple key-value filtering only
results = m.search(
    "query",
    filters={"user_id": "alice", "category": "work", "status": "active"}
)

# After (v1.0.0) - enhanced filtering with operators
results = m.search(
    "query",
    filters={
        "AND": [
            {"user_id": "alice"},
            {"category": "work"},
            {"status": {"ne": "archived"}},
            {"priority": {"gte": 5}}
        ]
    }
)
```
<注意>
  现有的平等过滤器继续工作；逐步添加新的运算符分支，以便代理可以采用更丰富的查询而无需停机。
</注>

## 查看实际效果

### 项目管理过滤```python
# Find high-priority active tasks
results = m.search(
    "What tasks need attention?",
    filters={
        "AND": [
            {"user_id": "project_manager"},
            {"project": {"in": ["alpha", ""]}},
            {"priority": {"gte": 8}},
            {"status": {"ne": "completed"}},
            {
                "OR": [
                    {"assignee": "alice"},
                    {"assignee": "bob"}
                ]
            }
        ]
    }
)
```
<信息图标=“检查”>
  返回的任务应属于目标项目，保持不完整，并分配给列出的队友之一。
</信息>

### 客户支持过滤```python
# Find recent unresolved tickets
results = m.search(
    "pending support issues",
    filters={
        "AND": [
            {"agent_id": "support_bot"},
            {"ticket_status": {"ne": "resolved"}},
            {"priority": {"in": ["high", "critical"]}},
            {"created_date": {"gte": "2024-01-01"}},
            {
                "NOT": [
                    {"category": "spam"}
                ]
            }
        ]
    }
)
```
<提示>
  将代理 ID 过滤器与特定于工单的元数据配对，以便共享支持机器人仅返回它们可以在当前会话中处理的工单。
</提示>

### 内容推荐过滤```python
# Personalized content filtering
results = m.search(
    "recommend content",
    filters={
        "AND": [
            {"user_id": "reader123"},
            {
                "OR": [
                    {"genre": {"in": ["sci-fi", "fantasy"]}},
                    {"author": {"contains": "favorite"}}
                ]
            },
            {"rating": {"gte": 4.0}},
            {"read_status": {"ne": "completed"}},
            {"language": "english"}
        ]
    }
)
```
<信息图标=“检查”>
  确认个性化源仅显示符合评级和语言标准的未读标题。
</信息>

### 处理无效运算符```python
try:
    results = m.search(
        "test query",
        filters={
            "user_id": "alice",
            "invalid_operator": {"unknown": "value"}
        }
    )
except ValueError as e:
    print(f"Filter error: {e}")
    results = m.search(
        "test query",
        filters={"user_id": "alice", "category": "general"}
    )
```
<警告>
  在执行搜索之前验证过滤器，以便您可以在开发期间而不是在运行时捕获拼写错误或不受支持的运算符。
</警告>

## 最佳实践

1. **首先使用索引字段：** 对过滤器进行排序，以便在复杂的字符串操作之前运行相等性检查。
2. **有意组合运算符：** 保持逻辑树可读 - 大嵌套更难调试。
3. **定期测试性能：** 使用类似生产的有效负载对关键查询进行基准测试。
4.**计划优雅的降级：**当操作员不可用时提供后备过滤器。
5. **尽早验证语法：** 在开发过程中捕获格式错误的过滤器，以在运行时保护代理。
