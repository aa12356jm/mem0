Mem0 镜像了 OpenAI 客户端界面，因此您可以通过最少的更改将内存插入现有的聊天完成代码中。将兼容 OpenAI 的客户端指向 Mem0，保持相同的请求形状，并在调用之间获得持久内存。

<信息>
  **你会在以下情况下使用它......**
  - 您的应用程序已经依赖于 OpenAI 聊天完成，并且您希望 Mem0 感觉熟悉。
  - 您需要重用需要 OpenAI 兼容响应的现有中间件。
  - 您计划在 Mem0 平台和自托管客户端之间切换，而无需重写代码。
</信息>

## 特点

- **插入客户端：** `client.chat.completions.create(...)` 与 OpenAI 的方法签名相同。
- **共享参数：** Mem0 接受 `messages`、`model` 和可选的内存范围字段（`user_id`、`agent_id`、`run_id`）。- **记忆感知响应：** 每次调用都会保存相关事实，以便将来的提示自动反映过去的对话。
- **OSS奇偶校验：** 无论您调用托管代理还是OSS配置，都使用相同的API接口。

<信息图标=“检查”>
  运行一个设置了“user_id”的请求。如果下一次调用引用该 ID 并且其回复使用存储的内存，则确认兼容性。
</信息>

---## 配置它

### 调用托管 Mem0 代理```python
from mem0.proxy.main import Mem0

client = Mem0(api_key="m0-xxx")

messages = [
    {"role": "user", "content": "I love Indian food but I cannot eat pizza since I'm allergic to cheese."}
]

chat_completion = client.chat.completions.create(
    messages=messages,
    model="gpt-5-mini",
    user_id="alice"
)
```
<提示>
  重复使用 OpenAI 客户端已发送的相同标识符，以便您可以在提供者之间切换，而无需分支逻辑。
</提示>

### 使用兼容OpenAI的OSS客户端```python
from mem0.proxy.main import Mem0

config = {
    "vector_store": {
        "provider": "qdrant",
        "config": {
            "host": "localhost",
            "port": 6333
        }
    }
}

client = Mem0(config=config)

chat_completion = client.chat.completions.create(
    messages=[{"role": "user", "content": "What's the capital of France?"}],
    model="gpt-5-mini"
)
```
## 查看实际效果

###记忆感知餐厅推荐```python
from mem0.proxy.main import Mem0

client = Mem0(api_key="m0-xxx")

# Store preferences
client.chat.completions.create(
    messages=[{"role": "user", "content": "I love Indian food but I'm allergic to cheese."}],
    model="gpt-5-mini",
    user_id="alice"
)

# Later conversation reuses the memory
response = client.chat.completions.create(
    messages=[{"role": "user", "content": "Suggest dinner options in San Francisco."}],
    model="gpt-5-mini",
    user_id="alice"
)

print(response.choices[0].message.content)
```
<信息图标=“检查”>
  第二个响应应该叫出印度餐馆并避免奶酪，证明 Mem0 召回了存储的偏好。
</信息>

## 验证该功能是否正常工作

- 比较 Mem0 与 OpenAI 的响应是否有相同的提示——两者都应返回相同的结构（“选择”、“用法”等）。
- 在每次请求后检查存储的内存，以确认事实提取捕获了正确的详细信息。
- 测试托管（`Mem0(api_key=...)`）和 OSS 配置之间的切换，以确保两者遵循相同的请求正文。

---

## 最佳实践

1. **有意限定上下文：** 仅当您希望对话持续时才传递标识符；跳过它们进行一次性呼叫。
2. **记录内存使用情况：** 检查“response.metadata.memories”（如果启用）以查看模型调用了哪些事实。
3.**重用中间件：** 将现有的 OpenAI 客户端包装器指向 Mem0 代理 URL，以避免代码漂移。
4. **处理后备：** 保留普通 OpenAI 调用的代码路径，以防 Mem0 不可用，然后稍后重新同步内存。

## 参数参考|参数|类型 |目的|
| ---| ---| ---|
| `用户 ID` | `str` |将对话与用户关联起来，以便保留记忆。 |
| `agent_id` | `str` |用于多代理场景的可选代理或机器人标识符。 |
| `run_id` | `str` |短期流的可选会话/运行标识符。 |
| `元数据` | `字典` |在每个内存条目旁边存储额外的字段。 |
| `过滤器` | `字典` |在做出反应时限制检索特定的记忆。 |
| `top_k` | `int` |限制 Mem0 拉入上下文的内存数量（默认 10）。 |

其他请求字段反映了 OpenAI 的聊天完成 API。
