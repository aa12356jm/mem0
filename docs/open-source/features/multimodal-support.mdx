多模式支持让 Mem0 从图像和常规文本中提取事实。添加屏幕截图、收据或产品照片，Mem0 会将见解存储为可搜索记忆，以便客服人员稍后可以调用它们。

<信息>
  **你会在以下情况下使用它......**
  - 用户共享屏幕截图、菜单或文档，并且您希望细节成为回忆。
  - 您已经收集了文本对话，但需要视觉上下文才能获得更好的答案。
  - 您需要一个能够处理 URL 和本地图像文件的工作流程。
</信息>

<警告>
  大于 20MB 的图像将被拒绝。在发送文件之前压缩文件或调整文件大小以避免错误。
</警告>

---

## 特征剖析

- **视觉处理：** Mem0 通过视觉模型运行图像，提取文本和关键细节。
- **内存创建：** 提取的信息存储为标准内存，以便搜索、过滤和分析继续工作。- **上下文链接：**同一对话中的视觉和文本轮流保持链接，为代理提供更丰富的上下文。
- **灵活的输入：**接受Python和JavaScript SDK中可公开访问的URL或base64编码的本地文件。<手风琴组>
  <Accordion title="支持的格式">
    |格式|用于 |笔记|
    | ---| ---| ---|
    | JPEG/JPG|照片和屏幕截图|相机捕获的默认选项。 |
    |巴布亚新几内亚 |具有透明度的图像 |保持清晰的文本和 UI 元素清晰。 |
    |网页版 |网页优化图像 |较小的有效负载可实现更快的上传。 |
    |动图 |静态或动画图形|适用于简单图形和短循环。 |
  </手风琴>
</手风琴组>

## 配置它

### 添加来自 URL 的图像消息

<代码组>```python Python
from mem0 import Memory

client = Memory()

messages = [
    {"role": "user", "content": "Hi, my name is Alice."},
    {
        "role": "user",
        "content": {
            "type": "image_url",
            "image_url": {
                "url": "https://example.com/menu.jpg"
            }
        }
    }
]

client.add(messages, user_id="alice")
```

```ts TypeScript
import { Memory } from "mem0ai";

const client = new Memory();

const messages = [
  { role: "user", content: "Hi, my name is Alice." },
  {
    role: "user",
    content: {
      type: "image_url",
      image_url: { url: "https://example.com/menu.jpg" }
    }
  }
];

await client.add(messages, { userId: "alice" });
```
</代码组>

<信息图标=“检查”>
  检查响应有效负载 - 内存列表应包括从菜单图像中提取的条目以及文本轮流。
</信息>

### 将本地图片上传为base64

<代码组>```python Python
import base64
from mem0 import Memory

def encode_image(image_path):
    with open(image_path, "rb") as image_file:
        return base64.b64encode(image_file.read()).decode("utf-8")

client = Memory()
base64_image = encode_image("path/to/your/image.jpg")

messages = [
    {
        "role": "user",
        "content": [
            {"type": "text", "text": "What's in this image?"},
            {
                "type": "image_url",
                "image_url": {
                    "url": f"data:image/jpeg;base64,{base64_image}"
                }
            }
        ]
    }
]

client.add(messages, user_id="alice")
```

```ts TypeScript
import fs from "fs";
import { Memory } from "mem0ai";

function encodeImage(imagePath: string) {
  const buffer = fs.readFileSync(imagePath);
  return buffer.toString("base64");
}

const client = new Memory();
const base64Image = encodeImage("path/to/your/image.jpg");

const messages = [
  {
    role: "user",
    content: [
      { type: "text", text: "What's in this image?" },
      {
        type: "image_url",
        image_url: {
          url: `data:image/jpeg;base64,${base64Image}`
        }
      }
    ]
  }
];

await client.add(messages, { userId: "alice" });
```
</代码组>

<提示>
  将 Base64 有效负载保持在 5MB 以下，以加快上传速度并避免达到 20MB 限制。
</提示>

### 错误处理

<代码组>```python Python
from mem0 import Memory
from mem0.exceptions import InvalidImageError, FileSizeError

client = Memory()

try:
    messages = [{
        "role": "user",
        "content": {
            "type": "image_url",
            "image_url": {"url": "https://example.com/image.jpg"}
        }
    }]

    client.add(messages, user_id="user123")
    print("Image processed successfully")

except InvalidImageError:
    print("Invalid image format or corrupted file")
except FileSizeError:
    print("Image file too large")
except Exception as exc:
    print(f"Unexpected error: {exc}")
```

```ts TypeScript
import { Memory } from "mem0ai";

const client = new Memory();

try {
  const messages = [{
    role: "user",
    content: {
      type: "image_url",
      image_url: { url: "https://example.com/image.jpg" }
    }
  }];

  await client.add(messages, { userId: "user123" });
  console.log("Image processed successfully");
} catch (error: any) {
  if (error.type === "invalid_image") {
    console.log("Invalid image format or corrupted file");
  } else if (error.type === "file_size_exceeded") {
    console.log("Image file too large");
  } else {
    console.log(`Unexpected error: ${error.message}`);
  }
}
```
</代码组>

<警告>
  快速失败无效格式，以便您可以提示用户在丢失上下文之前重新上传。
</警告>

## 验证该功能是否正常工作

- 调用“add”后，检查返回的内存并确认它们包含图像衍生的文本（菜单项、收据总数等）。
- 对图像中的细节进行后续“搜索”；记忆应该与相关文本一起出现。
- 监控图像上传延迟 - 大文件仍应在可接受的响应时间内完成。
- 记录文件大小和 URL 来源，以解决重复的故障。

---

## 最佳实践

1. **询问意图：** 提示用户解释为什么发送图像，以便记忆包含正确的上下文。
2. **保持图像可读：** 鼓励拍摄清晰的照片，不要使用大量滤镜或阴影，以便更好地提取。
3.**分割批量上传：**将多个图像作为单独的“add”调用发送，以隔离故障并提高可靠性。
4. **关注隐私：** 避免上传敏感文档，除非您的环境对于该数据是安全的。
5. **尽早验证文件大小：** 在编码之前检查文件大小以节省带宽和时间。

---

## 故障排除|问题 |原因 |修复 |
| ---| ---| ---|
|上传被拒绝 |文件大于 20MB |发送前压缩或调整大小。 |
|内存丢失图像数据|图像质量低或模糊 |使用更好的照明重新拍摄照片。 |
|无效格式错误 |不支持的文件类型 |首先转换为 JPEG 或 PNG。 |
|处理速度慢 |高分辨率图像 |缩小或压缩至 5MB 以下。 |
| Base64 错误 |前缀或编码不正确 |确保“data:image/<type>;base64,”存在且字符串有效。 |
