“AsyncMemory”为您提供了 Mem0 存储层的非阻塞接口，因此 Python 应用程序可以直接从异步代码添加、搜索和管理内存。当您将 Mem0 嵌入 FastAPI 服务、后台工作人员或任何依赖“asyncio”的工作流程时，请使用它。

<信息>
  **你会在以下情况下使用它......**
  - 您的代理已经在异步框架中运行，并且您需要内存调用来干净地等待。
  - 您想要在本地嵌入 Mem0 的存储，而不通过同步客户端发送请求。
  - 您计划将内存操作与其他异步 API（OpenAI、HTTP 调用、数据库）混合。
</信息>

<警告>
  `AsyncMemory` 需要一个正在运行的事件循环。始终在“async def”函数中或通过“asyncio.run()”等帮助程序调用它，以避免运行时错误。
</警告>

<注意>
  使用 TypeScript 工作？Node SDK 仍然使用同步调用——使用“Memory”，并在需要等待操作时依赖 Python 的“AsyncMemory”。
</注>

## 特征剖析- **直接存储访问：** `AsyncMemory` 与同步客户端相同的后端通信，但使所有内容保持在进程中以降低延迟。
- **方法奇偶校验：** 每个内存操作（“add”、“search”、“get_all”、“delete”等）都会镜像同步 API，让您可以重用有效负载形状。
- **并发执行：** 非阻塞 I/O 允许您使用“asyncio.gather”安排多个内存任务。
- **范围组织：** 继续使用 `user_id`、`agent_id` 和 `run_id` 来分隔会话和代理之间的内存。<手风琴组>
  <Accordion title="异步方法奇偶校验">
    |运营|异步签名 |笔记|
    | ---| ---| ---|
    |创造回忆| `等待内存.add(...)` |与同步“Memory.add”相同的参数。 |
    |搜索记忆 | `等待内存.search(...)` |返回带有“results”的字典，形状相同。 |
    |列出回忆 | `等待内存.get_all(...)` |按“user_id”、“agent_id”、“run_id”过滤。 |
    |找回记忆| `等待内存.get(memory_id=...)` |如果 ID 无效，则引发“ValueError”。 |
    |更新内存 | `等待内存.更新（内存_id = ...，数据= ...）` |接受部分更新。 |
    |删除记忆| `等待内存.删除(memory_id=...)` |返回确认负载。 |
    |批量删除 | `等待内存.delete_all(...)` |至少需要一个示波器过滤器。 |
    |历史| `等待内存.history(memory_id=...)` |获取更改日志以进行审核。|
  </手风琴>
</手风琴组>

## 配置它

### 初始化客户端```python
import asyncio
from mem0 import AsyncMemory

# Default configuration
memory = AsyncMemory()

# Custom configuration
from mem0.configs.base import MemoryConfig
custom_config = MemoryConfig(
    # Your custom configuration here
)
memory = AsyncMemory(config=custom_config)
```
<信息图标=“检查”>
  初始化后立即运行一次“await memory.search(...)”。如果它返回内存且没有错误，则您的配置有效。
</信息>

<提示>
  将配置对象保持在靠近异步客户端的位置，以便您可以在工作线程之间重用它们，而无需重新创建矢量存储连接。
</提示>

### 管理生命周期和并发性```python
import asyncio
from contextlib import asynccontextmanager
from mem0 import AsyncMemory

@asynccontextmanager
async def get_memory():
    memory = AsyncMemory()
    try:
        yield memory
    finally:
        # Clean up resources if needed
        pass

async def safe_memory_usage():
    async with get_memory() as memory:
        return await memory.search("test query", filters={"user_id": "alice"})
```
<提示>
  当您需要干净关闭时（例如，在 FastAPI 启动/关闭挂钩内），将客户端包装在异步上下文管理器中。
</提示>```python
async def batch_operations():
    memory = AsyncMemory()

    tasks = [
        memory.add(
            messages=[{"role": "user", "content": f"Message {i}"}],
            user_id=f"user_{i}"
        )
        for i in range(5)
    ]

    results = await asyncio.gather(*tasks, return_exceptions=True)
    for i, result in enumerate(results):
        if isinstance(result, Exception):
            print(f"Task {i} failed: {result}")
        else:
            print(f"Task {i} completed successfully")
```
<信息图标=“检查”>
  当并发正常工作时，成功的任务会返回内存 ID，而失败的任务会在“结果”列表中显示为异常。
</信息>

### 通过重试增加弹性```python
import asyncio
from mem0 import AsyncMemory

async def with_timeout_and_retry(operation, max_retries=3, timeout=10.0):
    for attempt in range(max_retries):
        try:
            return await asyncio.wait_for(operation(), timeout=timeout)
        except asyncio.TimeoutError:
            print(f"Timeout on attempt {attempt + 1}")
        except Exception as exc:
            print(f"Error on attempt {attempt + 1}: {exc}")

        if attempt < max_retries - 1:
            await asyncio.sleep(2 ** attempt)

    raise Exception(f"Operation failed after {max_retries} attempts")

async def robust_memory_search():
    memory = AsyncMemory()

    async def search_operation():
        return await memory.search("test query", filters={"user_id": "alice"})

    return await with_timeout_and_retry(search_operation)
```
<警告>
  始终限制重试 - 失控循环会使事件循环保持繁忙并阻塞其他任务。
</警告>

## 查看实际效果

### 核心运营```python
# Create memories
result = await memory.add(
    messages=[
        {"role": "user", "content": "I'm travelling to SF"},
        {"role": "assistant", "content": "That's great to hear!"}
    ],
    user_id="alice"
)

# Search memories
results = await memory.search(
    query="Where am I travelling?",
    filters={"user_id": "alice"}
)

# List memories
all_memories = await memory.get_all(filters={"user_id": "alice"})

# Get a specific memory
specific_memory = await memory.get(memory_id="memory-id-here")

# Update a memory
updated_memory = await memory.update(
    memory_id="memory-id-here",
    data="I'm travelling to Seattle"
)

# Delete a memory
await memory.delete(memory_id="memory-id-here")

# Delete scoped memories
await memory.delete_all(user_id="alice")
```
<信息图标=“检查”>
  确认每个调用返回与同步客户端相同的响应字段（ID、“结果”或确认对象）。缺少键通常意味着没有等待协程。
</信息>

<注意>
  “delete_all”至少需要“user_id”、“agent_id”或“run_id”之一。提供所有三个以将删除范围缩小到单个会话。
</注>

### 范围组织```python
await memory.add(
    messages=[{"role": "user", "content": "I prefer vegetarian food"}],
    user_id="alice",
    agent_id="diet-assistant",
    run_id="consultation-001"
)

all_user_memories = await memory.get_all(filters={"user_id": "alice"})
agent_memories = await memory.get_all(filters={"user_id": "alice", "agent_id": "diet-assistant"})
session_memories = await memory.get_all(filters={"user_id": "alice", "run_id": "consultation-001"})
specific_memories = await memory.get_all(
    filters={"user_id": "alice", "agent_id": "diet-assistant", "run_id": "consultation-001"}
)

history = await memory.history(memory_id="memory-id-here")
```
<提示>
  当您需要审核跟踪以确保合规性或调试更新逻辑时，请使用“历史记录”。
</提示>

### 与其他异步 API 混合```python
import asyncio
from openai import AsyncOpenAI
from mem0 import AsyncMemory

async_openai_client = AsyncOpenAI()
async_memory = AsyncMemory()

async def chat_with_memories(message: str, user_id: str = "default_user") -> str:
    search_result = await async_memory.search(query=message, filters={"user_id": user_id}, top_k=3)
    relevant_memories = search_result["results"]
    memories_str = "\n".join(f"- {entry['memory']}" for entry in relevant_memories)

    system_prompt = (
        "You are a helpful AI. Answer the question based on query and memories.\n"
        f"User Memories:\n{memories_str}"
    )

    messages = [
        {"role": "system", "content": system_prompt},
        {"role": "user", "content": message},
    ]

    response = await async_openai_client.chat.completions.create(
        model="gpt-5-mini",
        messages=messages
    )

    assistant_response = response.choices[0].message.content
    messages.append({"role": "assistant", "content": assistant_response})
    await async_memory.add(messages, user_id=user_id)

    return assistant_response
```
<信息图标=“检查”>
  当一切都正确连接时，OpenAI 响应应该包含最近的记忆，并且后续的“add”调用应该持续新的助手轮流。
</信息>

### 优雅地处理错误```python
from mem0 import AsyncMemory
from mem0.configs.base import MemoryConfig

async def handle_initialization_errors():
    try:
        config = MemoryConfig(
            vector_store={"provider": "chroma", "config": {"path": "./chroma_db"}},
            llm={"provider": "openai", "config": {"model": "gpt-5-mini"}}
        )
        AsyncMemory(config=config)
        print("AsyncMemory initialized successfully")
    except ValueError as err:
        print(f"Configuration error: {err}")
    except ConnectionError as err:
        print(f"Connection error: {err}")

async def handle_memory_operation_errors():
    memory = AsyncMemory()
    try:
        await memory.get(memory_id="non-existent-id")
    except ValueError as err:
        print(f"Invalid memory ID: {err}")

    try:
        await memory.search(query="", filters={"user_id": "alice"})
    except ValueError as err:
        print(f"Invalid search query: {err}")
```
<警告>
  捕获并记录来自无效输入的“ValueError”异常 - 否则异步堆栈跟踪可能会在后台任务中消失。
</警告>

### 通过 FastAPI 提供服务```python
from fastapi import FastAPI, HTTPException
from mem0 import AsyncMemory

app = FastAPI()
memory = AsyncMemory()

@app.post("/memories/")
async def add_memory(messages: list, user_id: str):
    try:
        result = await memory.add(messages=messages, user_id=user_id)
        return {"status": "success", "data": result}
    except Exception as exc:
        raise HTTPException(status_code=500, detail=str(exc))

@app.get("/memories/search")
async def search_memories(query: str, user_id: str, limit: int = 10):
    try:
        result = await memory.search(query=query, filters={"user_id": user_id}, top_k=limit)
        return {"status": "success", "data": result}
    except Exception as exc:
        raise HTTPException(status_code=500, detail=str(exc))
```
<提示>
  使用 FastAPI 时，为每个进程创建一个“AsyncMemory”实例 - 启动挂钩是配置和重用它的好地方。
</提示>

### 仪器记录```python
import logging
import time
from functools import wraps

logging.basicConfig(level=logging.INFO)
logger = logging.getLogger(__name__)

def log_async_operation(operation_name):
    def decorator(func):
        @wraps(func)
        async def wrapper(*args, **kwargs):
            start_time = time.time()
            logger.info(f"Starting {operation_name}")
            try:
                result = await func(*args, **kwargs)
                duration = time.time() - start_time
                logger.info(f"{operation_name} completed in {duration:.2f}s")
                return result
            except Exception as exc:
                duration = time.time() - start_time
                logger.error(f"{operation_name} failed after {duration:.2f}s: {exc}")
                raise
        return wrapper
    return decorator

@log_async_operation("Memory Add")
async def logged_memory_add(memory, messages, user_id):
    return await memory.add(messages=messages, user_id=user_id)
```
<信息图标=“检查”>
  记录的持续时间为您提供了在 AsyncMemory 投入生产后发现回归所需的基线。
</信息>

## 验证该功能是否正常工作

- 运行快速添加/搜索周期并确认返回的内存内容与您的输入匹配。
- 检查应用程序日志以确保异步任务完成而不阻塞事件循环。
- 在 FastAPI 或其他框架中，点击运行状况端点以验证共享客户端是否处理并发请求。
- 监控重试计数器——意外的峰值表明配置或连接问题。

## 最佳实践

1. **保持等待操作：** 忘记 `await` 是错过写入的最快方法 - 对其进行 lint 或添加辅助包装器。
2. **仔细删除范围：** 始终提供 `user_id`、`agent_id` 或 `run_id` 以避免清除太多数据。
3. **批量写入深思熟虑：** 使用 `asyncio.收集吞吐量，但根据后端容量限制并发性。
4. **使用上下文记录错误：** 捕获用户和代理范围以快速分类故障。
5. **重用客户端：** 每个工作进程实例化一次`AsyncMemory`以避免重复的后端握手。

## 故障排除|问题 |可能的原因 |修复 |
| ---| ---| ---|
|初始化失败 |缺少依赖项，无效配置 |验证“MemoryConfig”设置和环境变量。 |
|操作缓慢|大数据集、网络延迟 |缓存大量查询并调整向量存储参数。 |
|找不到内存 | ID无效或记录已删除 |检查 ID 来源并处理软删除状态。 |
|连接超时 |网络问题、后端超载 |应用重试/退避并检查基础设施运行状况。 |
|内存不足错误 |超大批量|将并发或块操作减少为更小的集合。 |
