# 配置Mem0 OSS组件

<信息>
  **先决条件**
  - Python 3.10+ 可用“pip”
  - 运行矢量数据库（例如，Qdrant、Postgres + pgvector）或托管商店的访问凭据
  - 您选择的 LLM、嵌入器和重新排序提供商的 API 密钥
</信息>

<提示>
  如果您仍然需要基本 CLI 和存储库，请从 <Link href="/open-source/python-quickstart">Python 快速入门</Link> 开始。
</提示>

## 安装依赖项

<标签>
  <标签标题=“Python”>
<步骤>
<Step title="安装Mem0 OSS">```bash
pip install mem0ai
```
</步骤>
<Step title="添加提供商 SDK（例如：Qdrant + OpenAI）">```bash
pip install qdrant-client openai
```
</步骤>
</步骤>
  </标签>
  <选项卡标题=“Docker Compose”>
<步骤>
<Step title="克隆存储库并复制撰写文件">```bash
git clone https://github.com/mem0ai/mem0.git
cd mem0/examples/docker-compose
```
</步骤>
<Step title="安装本地覆盖的依赖项">```bash
pip install -r requirements.txt
```
</步骤>
</步骤>
  </标签>
</标签>

## 定义你的配置

<标签>
  <标签标题=“Python”>
<步骤>
<Step title="创建配置字典">```python
from mem0 import Memory

config = {
    "vector_store": {
        "provider": "qdrant",
        "config": {"host": "localhost", "port": 6333},
    },
    "llm": {
        "provider": "openai",
        "config": {"model": "gpt-4.1-mini", "temperature": 0.1},
    },
    "embedder": {
        "provider": "vertexai",
        "config": {"model": "textembedding-gecko@003"},
    },
    "reranker": {
        "provider": "cohere",
        "config": {"model": "rerank-english-v3.0"},
    },
}

memory = Memory.from_config(config)
```
</步骤>
<Step title="将机密存储为环境变量">```bash
export QDRANT_API_KEY="..."
export OPENAI_API_KEY="..."
export COHERE_API_KEY="..."
```
</步骤>
</步骤>
  </标签>
  <选项卡标题=“config.yaml”>
<步骤>
<Step title="创建 `config.yaml` 文件">```yaml
vector_store:
  provider: qdrant
  config:
    host: localhost
    port: 6333

llm:
  provider: azure_openai
  config:
    api_key: ${AZURE_OPENAI_KEY}
    deployment_name: gpt-4.1-mini

embedder:
  provider: ollama
  config:
    model: nomic-embed-text

reranker:
  provider: zero_entropy
  config:
    api_key: ${ZERO_ENTROPY_KEY}
```
</步骤>
<Step title="运行时加载配置文件">```python
from mem0 import Memory

memory = Memory.from_config_file("config.yaml")
```
</步骤>
</步骤>
  </标签>
</标签>

<信息图标=“检查”>
  运行 `memory.add(["记住我在东京最喜欢的咖啡馆。"], user_id="alex")`，然后运行 `memory.search("最喜欢的咖啡馆",filters={"user_id": "alex"})`。您应该看到 Qdrant 集合已填充，并且重新排序器将内存标记为热门命中。
</信息>

## 调整组件设置

<手风琴组>
  <Accordion title="矢量商店集合">
    在生产中显式命名集合（“collection_name”）以隔离租户并启用每个租户的保留策略。
  </手风琴>
  <Accordion title="LLM 提取温度">
    保持提取温度 ≤0.2，以便高级存储器保持确定性。仅当您发现缺失的事实时才提高它。
  </手风琴>
  <Accordion title="Reranker 深度">
    将“top_k”限制为 10-20 个结果；发送更多会增加延迟，而没有任何有意义的收益。</手风琴>
</手风琴组>

<警告>
  混合托管和自托管组件？确保每个出站提供商呼叫都通过安全网络路径进行。即使您的矢量存储位于本地，托管重新排名程序通常也需要出站互联网。
</警告>

## 快速恢复- Qdrant 连接错误 → 确认端口“6333”已公开并且 API 密钥（如果设置）匹配。
- 清空搜索结果→验证嵌入器型号名称；不匹配会导致尺寸错误。
- `Unknown reranker` → 更新 SDK (`pip install --upgrade mem0ai`) 以加载最新的提供商注册表。
