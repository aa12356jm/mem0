# Mem0 开源概述

Mem0 开源提供与平台相同的自适应内存引擎，但为需要在自己的基础设施上运行所有内容的团队打包。您拥有堆栈、数据和定制。

## Mem0 OSS 提供什么

- **完全控制**：调整环境中的每个组件，从 LLM 到矢量存储。
- **离线就绪**：当合规性或隐私需要时，将内存保留在您自己的网络上。
- **可扩展代码库**：分叉存储库、添加提供程序并发布自定义自动化。

<信息>
  运行 Mem0 OSS 的两种方法：作为应用程序（Python 或 Node）内的**库**，或作为带有仪表板、每用户 API 密钥和请求审核日志的**自托管服务器**。
</信息>

## 默认组件

<注意>
  **库默认值**（当您“导入” Mem0 并直接调用“Memory()”时）：
  - 法学硕士：OpenAI `gpt-5-mini`（通过`OPENAI_API_KEY`）
  - 嵌入：OpenAI `text-embedding-3-small`
  - 矢量存储：位于 `/tmp/qdrant` 的本地 Qdrant
  - 历史存储：SQLite 位于 `~/.mem0/history.db`
  - Reranker：在配置之前禁用

  使用 <Link href="/open-source/configuration">`Memory.from_config`</Link> 覆盖任何组件。
</注><注意>
  **Self-hosted server defaults** (the `server/` Docker Compose stack):
  - LLM: OpenAI `gpt-4.1-nano-2025-04-14` (override with `MEM0_DEFAULT_LLM_MODEL`)
  - Embeddings: OpenAI `text-embedding-3-small` (override with `MEM0_DEFAULT_EMBEDDER_MODEL`)
  - 矢量存储：Postgres + pgvector
  - Bundled providers: `openai`, `anthropic`, `gemini` — switch from the Configuration page

  See <Link href="/open-source/setup#supported-providers">Self-Hosted Setup</Link> for the full provider list and how to extend it.
</注>
