# 自定义类别

Mem0 自动标记每个记忆，但默认标签（旅行、运动、音乐等）可能与您的应用使用的名称不匹配。自定义类别允许您替换该列表，以便标签与您自己的措辞对齐。

<信息>
  **当……时使用自定义类别**
  - 您需要 Mem0 用您的产品团队已经使用的名称来标记内存。
  - 您想要依赖这些标签的干净报告或自动化。
  - 您正在从开源版本迁移并希望在这里使用相同的标签。
</信息>

<警告>
  托管 API 尚不支持按请求覆盖（“client.add”上的“custom_categories=...”）。在项目级别设置类别，然后像往常一样摄取记忆。
</警告>

## 配置访问

- 确保在您的环境中设置了“MEM0_API_KEY”或将其传递给 SDK 构造函数。- 如果您将工作范围限定为特定组织/项目，请使用这些标识符初始化客户端。

## 它是如何工作的- **默认列表** — 每个项目都以 15 个大类开始，例如“旅行”、“运动”和“音乐”。
- **项目覆盖** — 当您调用 `project.update(custom_categories=[...])` 时，该列表将替换未来记忆的默认值。
- **自动标签** — 当新的记忆进入时，Mem0 会从列表中挑选最接近的匹配项并将它们保存在“类别”字段中。

<注意>
  默认目录：`个人详细信息`、`家庭`、`专业详细信息`、`运动`、`旅行`、`食物`、`音乐`、`健康`、`技术`、`爱好`、`时尚`、`娱乐`、`里程碑`、`用户偏好`、`杂项`。
</注>

## 配置它

### 1. 在项目级别设置自定义类别

<代码组>```python Code
import os
from mem0 import MemoryClient

os.environ["MEM0_API_KEY"] = "your-api-key"

client = MemoryClient()

# Update custom categories
new_categories = [
    {"lifestyle_management_concerns": "Tracks daily routines, habits, hobbies and interests including cooking, time management and work-life balance"},
    {"seeking_structure": "Documents goals around creating routines, schedules, and organized systems in various life areas"},
    {"personal_information": "Basic information about the user including name, preferences, and personality traits"}
]

response = client.project.update(custom_categories=new_categories)
print(response)
```

```json Output
{
    "message": "Updated custom categories"
}
```
</代码组>

### 2.确认活动目录

<代码组>```python Code
# Get current custom categories
categories = client.project.get(fields=["custom_categories"])
print(categories)
```

## 默认类别（后备）

如果您不执行任何操作，内存将使用下面的内置集进行标记。```
- personal_details
- family
- professional_details
- sports
- travel
- food
- music
- health
- technology
- hobbies
- fashion
- entertainment
- milestones
- user_preferences
- misc
```
<代码组>```python Code
import os
from mem0 import MemoryClient

os.environ["MEM0_API_KEY"] = "your-api-key"

client = MemoryClient()

messages = [
    {"role": "user", "content": "Hi, my name is Alice."},
    {"role": "assistant", "content": "Hi Alice, what sports do you like to play?"},
    {"role": "user", "content": "I love playing badminton, football, and basketball. I'm quite athletic!"},
    {"role": "assistant", "content": "That's great! Alice seems to enjoy both individual sports like badminton and team sports like football and basketball."},
    {"role": "user", "content": "Sometimes, I also draw and sketch in my free time."},
    {"role": "assistant", "content": "That's cool! I'm sure you're good at it."}
]

# Add memories with default categories
client.add(messages, user_id='alice')
```

```
</代码组>

## 验证该功能是否正常工作

- `client.project.get(["custom_categories"])` 返回您设置的类别列表。
- `client.get_all(filters={"user_id": ...})` 显示新内存中填充的`categories` 列表。
- Mem0 仪表板（项目→内存）在类别列中显示自定义标签。

## 最佳实践

- 保持类别描述简洁但具体；分类器使用它们来消除歧义。
- 使用空“类别”查看记忆，看看可以在哪里扩展或重命名您的列表。
- 坚持项目级别的覆盖，直到发布每个请求的支持；混合方法会导致混乱。
