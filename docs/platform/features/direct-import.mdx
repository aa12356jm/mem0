## 如何使用直接导入

直接导入功能允许用户跳过内存扣除阶段，直接将预定义的内存输入系统中进行存储和检索。要启用此功能，请在“add”方法中将“infer”参数设置为“False”。


<代码组>```python Python
messages = [
    {"role": "user", "content": "Alice loves playing badminton"},
    {"role": "assistant", "content": "That's great! Alice is a fitness freak"},
    {"role": "user", "content": "Alice mostly cooks at home because of her gym plan"},
]


client.add(messages, user_id="alice", infer=False)
```

```markdown Output
[]
```
</代码组>

您可以看到 add 调用的输出是一个空列表。

<注意>只有角色为“user”的消息才会被用于存储。具有“助理”、“系统”等角色的消息在存储过程中将被忽略。</Note>

<警告>
直接导入会跳过推理管道，因此也会跳过重复检测。如果您稍后使用“infer=True”发送相同的事实，Mem0 将存储第二个副本。除非您确实需要两个版本，否则为每个内存源选择一种模式。
</警告>

## 如何找回记忆

您可以使用“搜索”方法检索记忆。

<代码组>```python Python
client.search("What is Alice's favorite sport?", filters={"user_id": "alice"})
```

    {
      "id": "19d6d7aa-2454-4e58-96fc-e74d9e9f8dd1",

      "categories": null,

      "updated_at": "2024-10-15T21:52:11.474912-07:00"
    }
  ]
}
```
</代码组>

## 如何找回所有记忆

您可以使用“get_all”方法检索所有记忆。

<Callout type="warning" title="需要过滤器">
`get_all()` 现在需要指定过滤器。
</标注>

<代码组>```python Python
client.get_all(filters={"AND": [{"user_id": "alice"}]})
```
