> 内存过滤器提供了一种灵活的方式来查询和检索内存存储中的特定内存。您可以按用户、代理、内容类别、时间范围进行过滤，并使用逻辑运算符组合多个条件。

## 何时使用过滤器

使用大规模内存存储时，您需要精确控制要检索的内存。过滤器可以帮助您：

* **隔离用户数据**：检索特定用户的记忆，同时维护隐私
* **调试和审计**：导出特定内存子集进行分析
* **目标内容**：查找具有特定类别或元数据的记忆
* **基于时间的查询**：检索特定日期范围内的记忆
* **性能优化**：通过预过滤降低查询复杂度

## 过滤器结构

过滤器使用嵌套的 JSON 结构，根部带有逻辑运算符：```python
# Basic structure
{
    "AND": [  # or "OR", "NOT"
        { "field": "value" },
        { "field": { "operator": "value" } }
    ]
}
```
## 可用字段和运算符

### 实体字段
|领域|运营商|示例|
|--------|------------|---------|
| `用户 ID` | `eq`、`ne`、`in`、`*` | `{"user_id": "user_123"}` |
| `agent_id` | `eq`、`ne`、`in`、`*` | `{"agent_id": "*"}` |
| `app_id` | `eq`、`ne`、`in`、`*` | `{"app_id": {"in": ["app1", "app2"]}}` |
| `run_id` | `eq`、`ne`、`in`、`*` | `{"run_id": "*"}` |

### 时间字段
|领域|运营商|示例|
|--------|------------|---------|
| `创建于` | `gt`、`gte`、`lt`、`lte`、`eq`、`ne` | `{"created_at": {"gte": "2024-01-01"}}` |
| `updated_at` | `gt`、`gte`、`lt`、`lte`、`eq`、`ne` | `{"updated_at": {"lt": "2024-12-31"}}` |
| `时间戳` | `gt`、`gte`、`lt`、`lte`、`eq`、`ne` | `{"timestamp": {"gt": "2024-01-01"}}` |

### 内容字段
|领域|运营商|示例|
|--------|------------|---------|
| `类别` | `eq`、`ne`、`in`、`包含` | `{"categories": {"in": ["finance"]}}` |
| `元数据` | `eq`、`ne`、`包含` | `{"元数据": {"键": "值"}}` |
| `关键字` | `包含`、`图标` | `{"keywords": {"icontains": "发票"}}` |### 特殊字段
|领域|运营商|示例|
|--------|------------|---------|
| `内存_id` | `在` | `{"memory_ids": ["id1", "id2"]}` |

<Callout type="warning" icon="exclamation-triangle" color="#F7B731">
“*”通配符匹配任何非空值。该字段具有空值的记录将被排除。
</标注>

<标注类型=“信息”图标=“键盘”颜色=“#00A8FF”>
完全按照所示使用运算符关键字（“eq”、“ne”、“gte”等）。平台 API 会拒绝诸如“>=”或“!=”之类的 SQL 样式符号。
</标注>

## 常见的过滤模式

使用这些现成的过滤器来定位典型的检索场景，而无需从头开始重建逻辑。

<手风琴组>
  <Accordion title="单用户">    ```python
    # Narrow to one user's memories
    filters = {"AND": [{"user_id": "user_123"}]}
    memories = client.get_all(filters=filters)
    ```
</手风琴>

  <Accordion title="所有用户">    ```python
    # Wildcard skips null user_id entries
    filters = {"AND": [{"user_id": "*"}]}
    memories = client.get_all(filters=filters)
    ```
</手风琴>

  <Accordion title="所有运行中的用户">    ```python
    # Pair a user filter with a run wildcard
    filters = {
        "AND": [
            {"user_id": "user_123"},
            {"run_id": "*"}
        ]
    }
    memories = client.get_all(filters=filters)
    ```
</手风琴>
</手风琴组>

<Callout type="warning" icon="exclamation-triangle" color="#E74C3C">
元数据过滤器仅支持裸值/`eq`、`contains` 和 `ne`。 “in”、“gt”或“lt”等运算符会触发“FilterValidationError”。对于多值检查，请将多个相等子句包装在“OR”中。
</标注>```python
# Multi-value metadata workaround
filters = {
    "OR": [
        {"metadata": {"type": "semantic"}},
        {"metadata": {"type": "episodic"}}
    ]
}
```

### 多个条件

组合各种过滤器以实现跨不同维度的复杂查询。

## 最佳实践

<Callout type="tip" icon="lightbulb" color="#26A17B">
根必须是带有一系列条件的“AND”、“OR”或“NOT”。
</标注>

<Callout type="tip" icon="lightbulb" color="#26A17B">
使用“*”来匹配字段的任何非空值。
</标注>

<Callout type="warning" icon="exclamation-triangle" color="#E74C3C">
内存按实体（用户、代理、应用程序、运行）存储。在同一个“AND”子句中组合“user_id”**和**“agent_id”不会返回任何结果，因为没有记录同时包含这两个值。一次查询一个实体范围或使用“OR”逻辑进行并行查找。
</标注>

## 故障排除

<手风琴组>
  <Accordion title="缺少带有 agent_id 的结果">
    **问题**：按“user_id”过滤，但看不到代理记忆。

    **解决方案**：用户和代理记忆作为单独的记录存储。使用 OR 查询两个范围：    ```python
    {"OR": [{"user_id": "user_123"}, {"agent_id": "agent_name"}]}
    ```
</手风琴>

  <Accordion title="ne 运算符返回太多">
    **问题**：“ne”比较会拉入具有空值的记录。

    **解决方案**：将 `ne` 与通配符防护配对：    ```python
    {"AND": [{"agent_id": "*"}, {"agent_id": {"ne": "old_agent"}}]}
    ```
</手风琴>

  <Accordion title="不区分大小写的搜索">
    **解决方案**：交换到“icontains”以标准化大小写。
  </手风琴>

  <Accordion title="两个日期之间的日期范围">
    **解决方案**：使用 `gte` 作为开始边界，使用 `lt` 作为结束边界：    ```python
    {"AND": [
        {"created_at": {"gte": "2024-01-01"}},
        {"created_at": {"lt": "2024-02-01"}}
    ]}
    ```
</手风琴>

  <Accordion title="元数据过滤器不起作用">
  **解决方案**：完全匹配顶级元数据键：  ```python
  {"metadata": {"source": "email"}}
  ```
</手风琴>
</手风琴组>

## 常见问题解答

<手风琴组>
  <Accordion title="我需要和/或/不需要吗？">
    是的。根必须是带有数组的逻辑运算符。
  </手风琴>

  <Accordion title="* 匹配什么？">
    任何非空值。排除空值。
  </手风琴>

  <Accordion title="为什么使用通配符？">
    未指定的字段默认为 NULL。使用“*”来包含非空值。
  </手风琴>

  <Accordion title="= 是必需的吗？">
    不。平等是默认设置：`{"user_id": "u1"}` 有效。
  </手风琴>

  <Accordion title="我可以过滤嵌套元数据吗？">
    仅支持顶级密钥。
  </手风琴>

  <Accordion title="如何搜索文本？">
    将“keywords”与“contains”（区分大小写）或“icontains”（不区分大小写）结合使用。
  </手风琴>

<Accordion title="我可以嵌套 AND/OR 吗？">    ```python
    {
        "AND": [
            {"user_id": "user_123"},
            {"OR": [
                {"categories": "finance"},
                {"categories": "health"}
            ]}
        ]
    }
    ```
</手风琴>
</手风琴组>

## 已知限制

- 实体过滤器对每个记录的单个范围进行操作。使用单独的查询或“OR”逻辑来比较用户与代理。
- 元数据仅支持裸/`eq`、`contains` 和 `ne` 比较。
- 通配符 (`"*"` ) 仅匹配字段已非空的记录。
