## 概述

Webhook 可为 Mem0 项目中的内存事件启用实时通知。 Webhook 在项目级别进行配置，这意味着每个 Webhook 都与特定项目相关联，并且仅接收来自该项目的事件。您可以配置 Webhook，以便在创建、更新、删除或分类记忆时向您指定的 URL 发送 HTTP POST 请求。

## 管理 Webhooks

### 创建 Webhook

为您的项目创建一个 webhook。它将仅接收来自该项目的事件：
<代码组>```python Python
import os
from mem0 import MemoryClient

os.environ["MEM0_API_KEY"] = "your-api-key"

client = MemoryClient()

# Create webhook in a specific project
webhook = client.create_webhook(
    url="https://your-app.com/webhook",
    name="Memory Logger",
    project_id="proj_123",
    event_types=["memory_add", "memory_categorize"]
)
print(webhook)
```

```javascript JavaScript
const { MemoryClient } = require('mem0ai');
const client = new MemoryClient({ apiKey: 'your-api-key'});

// Create webhook in a specific project
const webhook = await client.createWebhook({
    url: "https://your-app.com/webhook",
    name: "Memory Logger",
    projectId: "proj_123",
    eventTypes: ["memory_add", "memory_categorize"]
});
console.log(webhook);
```

### 更新 Webhook

通过指定“webhook_id”来更新现有 Webhook 的配置：

<代码组>```python Python
# Update webhook for a specific project
updated_webhook = client.update_webhook(
    name="Updated Logger",
    url="https://your-app.com/new-webhook",
    event_types=["memory_update", "memory_add"],
    webhook_id="wh_123"
)
print(updated_webhook)
```

```javascript JavaScript
// Update webhook for a specific project
const updatedWebhook = await client.updateWebhook({
    name: "Updated Logger",
    url: "https://your-app.com/new-webhook",
    eventTypes: ["memory_update", "memory_add"],
    webhookId: "wh_123"
});
console.log(updatedWebhook);
```

```json Output
{
  "message": "Webhook updated successfully"
}
```
</代码组>

### 删除 Webhook

通过提供“webhook_id”来删除 Webhook：

<代码组>```python Python
# Delete webhook from a specific project
response = client.delete_webhook(webhook_id="wh_123")
print(response)
```

```javascript JavaScript
// Delete webhook from a specific project
const response = await client.deleteWebhook({webhookId: "wh_123"});

```

```json Output
{
  "message": "Webhook deleted successfully"

```
</代码组>

## 事件类型

Mem0 支持以下 Webhook 事件类型：

- `memory_add`：添加内存时触发。
- `memory_update`：更新现有内存时触发。
- `memory_delete`：删除内存时触发。
- `memory_categorize`：当内存被分类时触发。

## Webhook 负载

当发生内存事件时，Mem0 会向您的 Webhook URL 发送一个 HTTP POST 请求，其中包含以下负载：

**内存添加/更新/删除有效负载：**```json
{
    "event_details": {
        "id": "a1b2c3d4-e5f6-4g7h-8i9j-k0l1m2n3o4p5",
            "data": {
            "memory": "Name is Alex"
            },
        "event": "ADD"
    }
}
```
**内存分类有效负载：**```json
{
    "event_details": {
        "event": "CATEGORIZE",
        "memory_id": "a1b2c3d4-e5f6-4g7h-8i9j-k0l1m2n3o4p5",
        "categories": ["hobbies", "travel"]
    }
}
```
## 最佳实践

1. **实现重试逻辑**：确保您的 webhook 端点可以处理临时故障。
2. **验证 Webhook 来源**：实施安全措施以验证 Webhook 请求是否源自 Mem0。
3. **异步处理事件**：异步处理 webhook 事件以避免超时并确保可靠处理。
4. **监控 Webhook 运行状况**：定期检查您的 Webhook 日志，以确保功能正常并及时解决传送失败问题。
