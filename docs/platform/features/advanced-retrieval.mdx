## 什么是高级检索？

高级检索使您能够精确控制如何查找和排序记忆。虽然基本搜索使用语义相似性，但这些高级选项可帮助您在需要时准确找到所需内容。

## 搜索增强选项

### 重新排名

使用深度语义理解对结果重新排序，将最相关的记忆放在第一位。

<标签>
  <Tab title="何时使用">
- 需要将最相关的结果放在顶部
- 结果顺序对于您的应用至关重要
- 希望不同查询的质量保持一致
- 构建面向用户的功能，其中准确性很重要
</标签>
<标签标题=“它是如何工作的”>```python Python
# Get the most relevant travel plans first
results = client.search(
    query="What are my upcoming travel plans?",
    rerank=True,
    user_id="user123"
)

# Before reranking:        After reranking:
# 1. "Went to Paris"   →   1. "Tokyo trip next month"
# 2. "Tokyo trip next" →   2. "Need to book hotel in Tokyo"  
# 3. "Need hotel"      →   3. "Went to Paris last year"
```
</标签>
<选项卡标题=“性能”>
- **延迟**：额外 150-200 毫秒
- **准确性**：显着提高
- **排序**：更相关
- **最适合**：Top-N 精度、面向用户的结果
</标签>

## 最佳实践

### 做

- 从基本搜索开始简单并衡量影响，然后再启用重新排名
- 当最高结果质量最重要时使用重新排名
- 监控延迟并根据应用程序的需求进行调整
- 优雅地处理空结果

### 不要

- 默认情况下启用重新排名，无需衡量必要性
- 忽略实时应用程序中的延迟影响
- 使用高级检索来实现简单、快速的查找场景

## 性能指南

### 延迟预期```python Python
# Performance monitoring example
import time

start_time = time.time()
results = client.search(
    query="user preferences",
    rerank=True,         # +150ms
    user_id="user123"
)
latency = time.time() - start_time
print(f"Search completed in {latency:.2f}s")
```
### 优化技巧

1. **缓存频繁查询**以避免重复的高级处理
2. **使用特定于会话的搜索**和`run_id`来减少搜索空间
3. **当搜索返回空结果时实施回退逻辑**
4. **监视并警告**搜索延迟模式
