## 什么是情境记忆创建？

上下文记忆创建会自动管理消息历史记录，让您能够专注于构建 AI 体验，而无需手动跟踪交互。只需发送新消息，Mem0 就会自动处理上下文。

<代码组>```python Python
# Just send new messages - Mem0 handles the context
messages = [
    {"role": "user", "content": "I love Italian food, especially pasta"},
    {"role": "assistant", "content": "Great! I'll remember your preference for Italian cuisine."}
]

client.add(messages, user_id="user123")
```

```javascript JavaScript
// Just send new messages - Mem0 handles the context
const messages = [
    {"role": "user", "content": "I love Italian food, especially pasta"},
    {"role": "assistant", "content": "Great! I'll remember your preference for Italian cuisine."}
];

await client.add(messages, { userId: "user123", version: "v2" });
```
</代码组>

## 为什么要使用情境记忆创建？

- **简单**：仅发送新消息，无需手动历史记录跟踪
- **高效**：更小的有效负载和更快的处理速度
- **自动**：由 Mem0 处理的上下文管理
- **可靠**：没有丢失交互历史记录的风险
- **可扩展**：随着应用程序的增长无缝工作

## 组织策略

根据您的应用程序需求选择正确的方法：

### 用户级内存（仅`user_id`）

**最适合：** 个人偏好、个人资料信息、长期用户数据

<代码组>```python Python
# Persistent user memories across all interactions
messages = [
    {"role": "user", "content": "I'm allergic to nuts and dairy"},
    {"role": "assistant", "content": "I've noted your allergies for future reference."}
]

client.add(messages, user_id="user123")
# This allergy info will be available in ALL future interactions
```

```javascript JavaScript
// Persistent user memories across all interactions
const messages = [
    {"role": "user", "content": "I'm allergic to nuts and dairy"},
    {"role": "assistant", "content": "I've noted your allergies for future reference."}
];

</代码组>

### 会话特定内存（`user_id` + `run_id`）

**最适合：** 特定于任务的上下文、单独的交互线程、基于项目的会话

## 最佳实践

### ✅ 做
- **按上下文范围组织**：仅将“user_id”用于持久数据，为特定于会话的上下文添加“run_id”
- **让消息集中**在当前交互上
- **使用真实的交互流程进行测试**以确保上下文按预期工作

### ❌不要
- 发送重复的消息或交互历史记录
- 跳过诸如“user_id”或“run_id”之类的内存范围标识符
- 在同一应用程序中混合上下文和非上下文方法

## 故障排除

|问题 |解决方案 |
|--------|----------|
| **上下文不起作用** |确保每个调用使用相同的“user_id”/“run_id”组合；版本是自动的 |
| **检索到错误的上下文** |检查不同的交互主题是否需要单独的“run_id”值 |
| **缺少交互历史记录** |验证交互线程中的所有消息都使用相同的“user_id”和“run_id” |
| **太多不相关的上下文** |使用更具体的“run_id”值来分隔不同的交互类型 |
