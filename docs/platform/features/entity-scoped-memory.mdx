Mem0 的平台 API 可让您为不同的用户、代理和应用程序分离内存。通过使用正确的标识符标记每个写入和查询，您可以防止数据在它们之间混合，保持清晰的审计跟踪并控制数据保留。

</提示>

<信息>
  **您将在以下情况下使用它......**
  - 您为多个需要私人内存空间的客户运行助手
  - 不同的代理（例如计划者和评论家）需要针对同一用户的不同上下文
  - 会话应按自己的时间表到期，使调试和数据删除更加精确
</信息>


## 配置访问```python
from mem0 import MemoryClient

client = MemoryClient(api_key="m0-...")
```
调用“client.project.get()”来验证您的连接。它应该返回您的项目详细信息，包括“org_id”和“project_id”。如果收到 401 错误，请在 Mem0 仪表板中生成新的 API 密钥。

## 特征剖析

|尺寸|领域 |何时使用它 |示例值 |
| ----------- | ---------- | ------------------------------------------------ | ------------------- |
|用户 | `用户 ID` |持久角色或帐户| `“customer_6412”` |
|代理| `agent_id` |独特的代理角色或工具| `“膳食计划”` |
|应用 | `app_id` |白标应用程序或产品表面 | `“ios_retail_demo”` |
|会议| `run_id` |短暂的流程、票证或对话线程 | `“ticket-9241”` |- **写入** (`client.add`) 接受这些字段的任意组合。缺失字段默认为“null”。
- **读取**（`client.search`、`client.get_all`、导出、删除）接受 `filters` JSON 对象内的相同标识符。
- **隐式 null 范围**：仅传递 `{"user_id": "alice"}` 会自动将结果限制为 `agent_id`、`app_id` 和 `run_id` 为 `null` 的记录。当您需要更广泛的联接时，添加通配符（“*”）、显式列表或其他过滤器。

<警告>
  **常见陷阱**：如果您使用 `user_id="alice"` 创建内存，但其他字段默认为 `null`，则使用 `{"AND": [{"user_id": "alice"}, {"agent_id": "bot"}]}` 搜索将不会返回任何内容，因为您正在寻找 `agent_id="bot"` 的内存，而不是 `null`。
</警告>

## 选择正确的标识符|标识符 |目的|示例用例 |
|------------|---------|--------------------|
| `用户 ID` |存储随处跟踪一个人的偏好、个人资料详细信息和历史行为 |饮食限制、座位偏好、会议习惯 |
| `agent_id` |将代理商的个性、运营模式或品牌声音保留在一处 |旅行社、礼宾员、客户支持角色 | 旅行社
| `app_id` |标记来自合作伙伴应用程序或部署的每个写入以进行租户分离 |白标部署、合作伙伴集成 |
| `run_id` |隔离应独立重置或过期的临时流 |支持票证、聊天会话、实验 |

有关更详细的示例，请参阅《按实体划分内存》手册。

## 配置它

下面的示例添加带有实体标签的记忆：```python
messages = [
    {"role": "user", "content": "I teach ninth-grade algebra."},
    {"role": "assistant", "content": "I'll tailor study plans to algebra topics."}
]

client.add(
    messages,
    user_id="teacher_872",
    agent_id="study_planner",
    app_id="district_dashboard",
    run_id="prep-period-2025-09-02"
)
```
响应将包括一个或多个内存 ID。检查仪表板 → 内存以确认该条目显示在正确的用户、代理、应用程序和运行下。

<警告>
包含“user_id”和“agent_id”（或其他组合）的平台写入将作为每个实体的单独记录进行持久保存，以便我们可以强制实施隐私边界。每条记录只包含一个主要实体，这就是为什么 `{"AND": [{"user_id": ...}, {"agent_id": ...}]}` 永远不会返回结果的原因。计划每个实体范围的搜索或使用“OR”组合范围。
</警告>

HTTP 等效项在 JSON 正文中使用具有相同标识符的“POST /v1/memories/”。有关 REST 详细信息，请参阅添加内存 API 参考。

## 查看实际效果

**1.存储范围内存**```python
traveler_messages = [
    {"role": "user", "content": "I prefer boutique hotels and avoid shellfish."},
    {"role": "assistant", "content": "Logged your travel preferences for future itineraries."}
]

client.add(
    traveler_messages,
    user_id="customer_6412",
    agent_id="travel_planner",
    app_id="concierge_portal",
    run_id="itinerary-2025-apr",
    metadata={"category": "preferences"}
)
```
**2.按用户范围检索**```python
user_scope = {
    "AND": [
        {"user_id": "customer_6412"},
        {"app_id": "concierge_portal"},
        {"run_id": "itinerary-2025-apr"}
    ]
}

user_results = client.search("Any dietary flags?", filters=user_scope)
print(user_results)
```
**3.按代理范围检索**```python
agent_scope = {
    "AND": [
        {"agent_id": "travel_planner"},
        {"app_id": "concierge_portal"}
    ]
}

agent_results = client.search("Any dietary flags?", filters=agent_scope)
print(agent_results)
```
<提示图标=“指南针”>
写入可以包含多个标识符，但搜索一次只能解析一个实体空间。查询给定调用中的用户范围*或*代理范围 - 组合两者今天会返回一个空列表。
</提示>

<提示图标=“闪闪发光”>
想要尝试 AND/OR 逻辑、嵌套运算符或通配符？ <Link href="/platform/features/v2-memory-filters">内存过滤器 v2 指南</Link>通过工作示例逐步介绍了每个过滤器模式。
</提示>

**4.审核应用程序的所有内容**```python
app_scope = {
    "AND": [
        {"app_id": "concierge_portal"}
    ],
    "OR": [
        {"user_id": "*"},
        {"agent_id": "*"}
    ]
}

page = client.get_all(filters=app_scope, page=1, page_size=20)
```
<信息>
通配符（“*”）仅包含非空值。当您需要“任何代理”或“任何用户”而不将结果限制为仅空记录时，请使用它们。
</信息>

**5.清理会话**```python
client.delete_all(
    user_id="customer_6412",
    run_id="itinerary-2025-apr"
)
```
<信息图标=“检查”>
成功删除会返回 `{"message": "Memories脸删除成功！"}`。再次运行之前的“get_all”调用以确认会话内存已删除。
</信息>

## 验证该功能是否正常工作

- 使用过滤器运行“client.search”，并确认仅出现预期的记忆。不匹配的标识符通常意味着范围界定中存在拼写错误。
- 检查 Mem0 仪表板过滤丸。用户、代理、应用程序和运行都应显示内存条目的填充值。
- 使用唯一的“run_id”调用“client.delete_all”，并确认其他会话保持不变（“get_all”中的计数应该只针对该运行而下降）。## 最佳实践

- 使用一致的标识符格式（例如“team-alpha”或“app-ios-retail”），以便您稍后可以查询或删除整个组
- 调试时，在每次调用之前打印过滤器，以验证通配符 (`"*"`)、列表和运行 ID 拼写是否正确
- 将实体过滤器与元数据过滤器（类别、created_at）相结合，以进行精确导出或审核
- 使用“run_id”进行临时会话，例如支持票证或实验，然后安排清理作业来删除它们
