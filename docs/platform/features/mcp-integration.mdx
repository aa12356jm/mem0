> 模型上下文协议 (MCP) 为 AI 代理提供了一种标准化方法，通过 Mem0 管理自己的内存，无需手动 API 调用。

## 为什么使用 MCP

在构建人工智能应用程序时，内存管理通常需要手动集成。 MCP 通过以下方式消除了这种复杂性：

- **通用兼容性**：适用于任何 MCP 兼容客户端（Claude、Claude Code、Cursor、Windsurf、VS Code、OpenCode）
- **代理自主**：人工智能代理决定何时保存、搜索或更新记忆
- **零基础设施**：无需维护服务器 - Mem0 的云 MCP 处理一切
- **标准化协议**：一种集成适用于您的所有人工智能工具

## 设置

使用单个命令将 Mem0 MCP 添加到所有支持的客户端：```bash
npx mcp-add \
  --name mem0-mcp \
  --type http \
  --url "https://mcp.mem0.ai/mcp" \
  --clients "claude,claude code,cursor,windsurf,vscode,opencode"
```
或者配置特定的客户端：```bash
npx mcp-add \
  --name mem0-mcp \
  --type http \
  --url "https://mcp.mem0.ai/mcp" \
  --clients "cursor"
```
对于手动配置，请将其添加到您的 MCP 客户端配置中：```json
{
  "mcpServers": {
    "mem0-mcp": {
      "type": "http",
      "url": "https://mcp.mem0.ai/mcp"
    }
  }
}
```
有关每个客户端的详细说明，请参阅 [Mem0 MCP 快速入门](/platform/mem0-mcp)。

## 可用工具

MCP 服务器向您的 AI 客户端公开 11 种内存工具：

|工具|目的|
|------|---------|
| `添加内存` |存储对话或事实 |
| `搜索记忆` |使用过滤器查找相关记忆 |
| `获取记忆` |用分页列出回忆 |
| `更新内存` |修改现有内存内容 |
| `删除内存` |删除特定记忆 |
| `删除所有记忆` |批量删除记忆|
| `删除实体` |删除用户/代理/应用程序实体 |
| `获取内存` |通过ID检索单个内存|
| `列表实体` |查看存储的实体 |
| `列表事件` |使用过滤器和分页列出内存操作事件 |
| `获取事件状态` |通过`event_id`检查异步内存操作的状态 |

## 它是如何工作的1. **配置 MCP 服务器** - 使用上面的设置命令将 Mem0 MCP 添加到您的 AI 客户端
2. **代理连接** - 您的 AI 客户端通过 HTTP 连接到 Mem0 的云 MCP 服务器
3. **自主记忆** - 代理决定何时存储/检索记忆作为其推理的一部分
4. **无需手动 API 调用** - 代理通过 MCP 工具自动管理内存

## 交互示例

连接后，您的 AI 代理可以：```
User: Remember that I'm allergic to peanuts
Agent: [calls add_memory] Got it! I've saved your peanut allergy.

User: What dietary restrictions do I know about?
Agent: [calls search_memories] You have a peanut allergy.
```
代理根据上下文自动决定何时使用记忆工具。

## 你可以做什么

Mem0 MCP 服务器为您的 AI 应用程序提供强大的内存功能：

- **健康追踪**：“我对花生和贝类过敏” - 添加新的健康信息
- **研究数据**：“存储这些试验参数：200 名参与者，双盲，安慰剂对照” - 保存结构化数据
- **偏好查询**：“你对我的饮食偏好了解多少？” - 搜索并检索相关记忆
- **项目更新**：“更新我的项目状态：移动应用程序现已完成 80%” - 修改现有内存
- **数据清理**：“删除2023年以来的所有记忆” - 批量删除过时的信息
- **主题概述**：“向我展示有关凤凰计划的所有内容” - 列出某个主题的所有记忆
