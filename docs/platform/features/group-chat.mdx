## 概述

群聊功能使 Mem0 能够处理涉及多个参与者的对话，并自动将记忆分配给各个发言者。这样可以精确跟踪每个参与者在协作讨论、团队会议或多代理对话中的偏好、特征和贡献。

当您提供带有参与者姓名的消息时，Mem0 会自动：
- 分别从每个参与者的消息中提取记忆
- 使用其名称作为“user_id”或“agent_id”将每个记忆归因于正确的说话者
- 维护每个参与者的个人记忆档案

## 群聊如何运作

当消息包含“name”字段时，Mem0 会自动检测群聊场景：```json
{
  "role": "user",
  "name": "Alice",
  "content": "Hey team, I think we should use React for the frontend"
}
```
当名称存在时，Mem0：
- 将消息格式化为“Alice（用户）：内容”以进行处理
- 提取记忆并正确归因于每个说话者
- 将发言者姓名存储为“user_id”（对于用户）或“agent_id”（对于助理/代理）

### 内存归属规则

- **用户消息**：“name”字段成为存储内存中的“user_id”
- **助理/座席消息**：“name”字段成为存储内存中的“agent_id”
- **没有名称的消息**：使用角色作为标识符回退到标准处理

## 使用群聊

### 基本群聊

添加多人对话的记忆：

<代码组>```python Python
from mem0 import MemoryClient

client = MemoryClient(api_key="your-api-key")

# Group chat with multiple users
messages = [
    {"role": "user", "name": "Alice", "content": "Hey team, I think we should use React for the frontend"},
    {"role": "user", "name": "Bob", "content": "I disagree, Vue.js would be better for our use case"},
    {"role": "user", "name": "Charlie", "content": "What about considering Angular? It has great enterprise support"},
    {"role": "assistant", "content": "All three frameworks have their merits. Let me summarize the pros and cons of each."}
]

response = client.add(
    messages,
    run_id="group_chat_1",
    infer=True
)
print(response)
```

### 获取会话的所有记忆

检索特定群聊会话的所有记忆：

<代码组>```python Python

    "AND": [
        {"user_id": "*"},
        {"run_id": "group_chat_1"}
    ]
}

all_memories = client.get_all(filters=filters, page=1)
print(all_memories)

## 异步模式支持

群聊支持异步处理以提高性能。默认情况下，内存添加是异步处理的。

<代码组>```python Python
# Group chat — async processing is the default
response = client.add(
    messages,
    run_id="groupchat_async",
    infer=True,
)
print(response)
```
</代码组>

## 消息格式要求

### 必填字段

群聊中的每条消息必须包含：

- `role`：参与者的角色（`"用户"`、`"助理"`、`"代理"`）
- `content`：消息内容
- `name`：参与者的姓名（群聊检测所需）

### 消息结构示例```json
{
  "role": "user",
  "name": "Alice",
  "content": "I think we should use React for the frontend"
}
```
### 支持的角色

- **`user`**：人类参与者（用`user_id`存储的记忆）
- **`assistant`**：AI助手（使用`agent_id`存储的记忆）

## 最佳实践

1. **一致的命名**：在各个会话中为参与者使用一致的名称，以保持正确的内存归属。

2. **明确的角色分配**：确保每个参与者都有正确的角色（“用户”、“助理”或“代理”），以进行正确的内存分类。

3. **会话管理**：使用有意义的“run_id”值来组织群聊会话并轻松检索。

4. **内存过滤**：在需要时使用过滤器从特定参与者或会话中检索内存。

5. **异步处理**：默认情况下，内存添加是异步处理的，这对于大型群组对话来说是理想的选择。

6.**搜索上下文**：利用搜索功能在群聊上下文中查找特定信息。

## 用例- **团队会议**：跟踪各个团队成员的偏好和贡献
- **客户支持**：为不同的客户维护单独的内存配置文件
- **多代理系统**：管理与多个人工智能助手的对话
- **协作项目**：跟踪个人偏好和专业知识领域
- **小组讨论**：维护每个参与者观点的背景
