Mem0 的反馈机制允许您提供有关应用程序生成的内存的反馈。该反馈用于提高记忆和搜索结果的准确性。

## 它是如何工作的

反馈机制是一个简单的 API，允许您提供有关应用程序生成的内存的反馈。反馈存储在数据库中并用于提高记忆和搜索结果的准确性。随着时间的推移，Mem0 不断从这些反馈中学习，改进其内存生成和搜索功能，以获得更好的性能。

## 提供反馈

您可以通过调用 Mem0 客户端上的“feedback”方法来提供有关内存的反馈。

<代码组>```python Python
from mem0 import MemoryClient

client = MemoryClient(api_key="your_api_key")

client.feedback(memory_id="your-memory-id", feedback="NEGATIVE", feedback_reason="I don't like this memory because it is not relevant.")
```

```javascript JavaScript
import MemoryClient from 'mem0ai';

const client = new MemoryClient({ apiKey: 'your-api-key'});

client.feedback({
    memory_id: "your-memory-id", 
    feedback: "NEGATIVE", 
    feedback_reason: "I don't like this memory because it is not relevant."
})
```
</代码组>

## 反馈类型

`feedback` 参数可以是以下值之一：

- `积极`：记忆是有用的。
- `NEGATIVE`：内存没有用。
- `VERY_NEGATIVE`：内存根本没用。

## 参数

`feedback` 方法接受以下参数：

|参数|类型 |必填 |描述 |
|------------|------|----------|----------|
| `内存_id` |字符串|是的 |反馈内存的ID |
| `反馈` |字符串|没有 |反馈类型：“积极”、“消极”或“非常消极”|
| `反馈原因` |字符串|没有 |反馈的可选解释 |

<注意>
将“None”或“null”传递给“feedback”和“feedback_reason”参数以删除内存的现有反馈。
</注>

## 批量反馈操作

对于具有大量反馈的应用程序，您可以一次提供多个内存的反馈：

<代码组>```python Python
from mem0 import MemoryClient

client = MemoryClient(api_key="your_api_key")

# Bulk feedback example
feedback_data = [
    {
        "memory_id": "memory-1", 
        "feedback": "POSITIVE", 
        "feedback_reason": "Accurately captured the user's preference"
    },
    {
        "memory_id": "memory-2", 
        "feedback": "NEGATIVE", 
        "feedback_reason": "Contains outdated information"
    }
]

for item in feedback_data:
    client.feedback(**item)
```

```javascript JavaScript
import MemoryClient from 'mem0ai';

const client = new MemoryClient({ apiKey: 'your-api-key'});

// Bulk feedback example
const feedbackData = [
    {
        memory_id: "memory-1", 
        feedback: "POSITIVE", 
        feedback_reason: "Accurately captured the user's preference"
    },
    {
        memory_id: "memory-2", 
        feedback: "NEGATIVE", 
        feedback_reason: "Contains outdated information"
    }
];

for (const item of feedbackData) {
    await client.feedback(item);
}
```
</代码组>

## 最佳实践

### 何时提供反馈

- 记忆检索后，您可以立即评估相关性
- 在用户交互过程中，当用户明确表示满意或不满意时
- 通过使用应用程序的成功指标进行自动评估

### 有效反馈原因

提供具体的、可操作的反馈原因：

**好的例子：**
- “包含过时的联系信息”
- “准确捕捉用户的饮食限制”
- “与当前对话上下文无关”

**避免模糊的原因：**
——《记性不好》
- “错”
- “不好”

### 反馈策略

1. 保持一致：对相似的记忆应用相同的标准
2.具体：详细的原因有助于更快地改进系统
3.监控模式：定期反馈分析有助于识别改进领域

## 错误处理

处理提交反馈时可能出现的错误：

<代码组>```python Python
from mem0 import MemoryClient
from mem0.exceptions import MemoryNotFoundError, APIError

client = MemoryClient(api_key="your_api_key")

try:
    client.feedback(
        memory_id="memory-123", 
        feedback="POSITIVE", 
        feedback_reason="Helpful context for user query"
    )
    print("Feedback submitted successfully")
except MemoryNotFoundError:
    print("Memory not found")
except APIError as e:
    print(f"API error: {e}")
except Exception as e:
    print(f"Unexpected error: {e}")
```

```javascript JavaScript
import MemoryClient from 'mem0ai';

const client = new MemoryClient({ apiKey: 'your-api-key'});

try {
    await client.feedback({
        memory_id: "memory-123", 
        feedback: "POSITIVE", 
        feedback_reason: "Helpful context for user query"
    });
    console.log("Feedback submitted successfully");
} catch (error) {
    if (error.status === 404) {
        console.log("Memory not found");
    } else {
        console.log(`Error: ${error.message}`);
    }
}
```
</代码组>

## 反馈分析

通过监控一段时间内的内存性能来跟踪反馈的影响。考虑实施：

- 反馈完成率：有多少百分比的记忆收到反馈
- 反馈分配：正反馈与负反馈的平衡
- 内存质量趋势：如何通过反馈量提高准确性
- 用户满意度指标：反馈与用户体验之间的相关性
