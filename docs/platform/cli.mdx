mem0 CLI 允许您直接从终端添加、搜索、列出、更新和删除内存。它与 Mem0 平台 API 配合使用，并以 npm 包和 Python 包的形式提供。

两种实现都提供相同的行为 - 相同的命令、相同的选项、相同的输出格式。

<提示>
**为 AI 代理构建。** 在任何命令上将 `--agent` （或 `--json`） 作为全局标志传递，以获得针对编程使用而优化的结构化 JSON 输出 — 已清理的字段、无颜色或旋转器以及 JSON 格式的错误。将其放入任何代理工具循环中，并进行零额外解析。
</提示>

## 安装

<代码组>```bash npm
npm install -g @mem0/cli
```

```bash pip
pip install mem0-cli
```
</代码组>

## 身份验证

运行交互式设置向导来配置您的 API 密钥：```bash
mem0 init
```
这会提示您输入 API 密钥和默认用户 ID、验证连接并在本地保存配置。

对于 CI/CD 或非交互式环境，将两个值作为标志传递：```bash
mem0 init --api-key m0-xxx --user-id alice
```
您还可以通过环境变量设置 API 密钥：```bash
export MEM0_API_KEY="m0-xxx"
```
## 快速开始```bash
# Add a memory
mem0 add "I prefer dark mode and use vim keybindings" --user-id alice

# Search memories
mem0 search "What are Alice's preferences?" --user-id alice

# List all memories for a user
mem0 list --user-id alice

# Get a specific memory
mem0 get <memory-id>

# Update a memory
mem0 update <memory-id> "I prefer light mode now"

# Delete a memory
mem0 delete <memory-id>
```
## 命令

### `mem0 初始化`

交互式设置向导。提示输入 API 密钥和默认用户 ID。```bash
mem0 init
mem0 init --api-key m0-xxx --user-id alice
mem0 init --email alice@company.com
```
如果检测到现有配置，CLI 将在覆盖之前要求确认。使用 `--force` 跳过提示（在 CI/CD 管道中很有用）。```bash
mem0 init --api-key m0-xxx --user-id alice --force
```
|旗帜|描述 |
|------|-------------|
| `--api-key` | API 密钥（跳过提示）|
| `-u, --user-id` |默认用户 ID（跳过提示）|
| `--电子邮件` |邮箱验证码登录 |
| `--代码` |验证码（与“--email”一起使用进行非交互式登录）|
| `--force` |无需确认即可覆盖现有配置 |

### `mem0 添加`

从文本、JSON 消息数组、文件或标准输入添加内存。```bash
mem0 add "I prefer dark mode" --user-id alice
mem0 add --file conversation.json --user-id alice
echo "Loves hiking on weekends" | mem0 add --user-id alice
```
|旗帜|描述 |
|------|-------------|
| `-u, --user-id` |范围到用户|
| `--agent-id` |代理范围 |
| `--消息` | JSON 格式的对话消息 |
| `-f, --file` |从 JSON 文件读取消息 |
| `-m，--元数据` | JSON 格式的自定义元数据 |
| `--类别` |类别（JSON 数组或逗号分隔）|
| `--graph / --no-graph` |启用或禁用图形内存提取 |
| `-o, --output` |输出格式：`text`、`json`、`quiet` |

### `mem0 搜索`

使用自然语言搜索记忆。```bash
mem0 search "dietary restrictions" --user-id alice
mem0 search "preferred tools" --user-id alice --output json --top-k 5
```
|旗帜|描述 |
|------|-------------|
| `-u, --user-id` |按用户过滤 |
| `-k, --top-k` |结果数（默认：10）|
| `--阈值` |最小相似度分数（默认值：0.3）|
| `--重新排名` |启用重新排名 |
| `--关键字` |使用关键字搜索而不是语义 |
| `--过滤器` |高级过滤表达式（JSON）|
| `--graph / --no-graph` |在搜索中启用或禁用图表 |
| `-o, --output` |输出格式：`text`、`json`、`table` |

### `mem0 列表`

使用可选的过滤器和分页列出记忆。```bash
mem0 list --user-id alice
mem0 list --user-id alice --category preferences --output json
mem0 list --user-id alice --after 2024-01-01 --page-size 50
```
|旗帜|描述 |
|------|-------------|
| `-u, --user-id` |按用户过滤 |
| `--页` |页码（默认：1）|
| `--页面大小` |每页结果（默认：100）|
| `--类别` |按类别过滤 |
| `--之后` |创建于日期 (YYYY-MM-DD) | 之后
| `--之前` |创建于日期 (YYYY-MM-DD) 之前 |
| `-o, --output` |输出格式：`text`、`json`、`table` |

### `mem0 获取`

通过ID检索特定内存。```bash
mem0 get 7b3c1a2e-4d5f-6789-abcd-ef0123456789
mem0 get 7b3c1a2e-4d5f-6789-abcd-ef0123456789 --output json
```
### `mem0 更新`

更新现有内存的文本或元数据。```bash
mem0 update <memory-id> "Updated preference text"
mem0 update <memory-id> --metadata '{"priority": "high"}'
echo "new text" | mem0 update <memory-id>
```
### `mem0 删除`

删除单个内存、某个范围的所有内存或整个实体。```bash
# Delete a single memory
mem0 delete <memory-id>

# Delete all memories for a user
mem0 delete --all --user-id alice --force

# Delete all memories project-wide
mem0 delete --all --project --force

# Preview what would be deleted
mem0 delete --all --user-id alice --dry-run
```
|旗帜|描述 |
|------|-------------|
| `--全部` |删除所有与范围过滤器匹配的记忆|
| `--实体` |删除该实体及其所有记忆|
| `--项目` |使用 `--all`：删除项目范围内的所有内存 |
| `--试运行` |预览而不删除 |
| `--force` |跳过确认提示 |

### `mem0 导入`

从 JSON 文件批量导入内存。```bash
mem0 import data.json --user-id alice
```
该文件应该是一个 JSON 数组，其中每个项目都有一个“内存”（或“文本”或“内容”）字段和可选的“user_id”、“agent_id”和“元数据”字段。

### `mem0 配置`

查看或修改本地 CLI 配置。```bash
mem0 config show              # Display current config (secrets redacted)
mem0 config get api_key       # Get a specific value
mem0 config set user_id bob   # Set a value
```
### `mem0 实体`

列出或删除实体（用户、代理、应用程序、运行）。```bash
mem0 entity list users
mem0 entity list agents --output json
mem0 entity delete --user-id alice --force
```
### `mem0 事件`

检查异步操作创建的后台处理事件（例如批量删除、大型添加作业）。```bash
# List recent events
mem0 event list

# Check the status of a specific event
mem0 event status <event-id>
```
|旗帜|描述 |
|------|-------------|
| `-o, --output` |输出格式：`text`、`json` |

### `mem0 状态`

验证您的 API 连接并显示当前项目。```bash
mem0 status
```
### `mem0 版本`

打印 CLI 版本。```bash
mem0 version
```
## 输出格式

所有命令都支持“--output”标志来控制结果的显示方式：

|格式|描述 |
|--------|-------------|
| `文本` |具有颜色和格式的人类可读输出（大多数命令的默认设置）|
| `json` |结构化 JSON，适合通过管道传输到“jq”或由 AI 代理使用 |
| `表` |表格格式（默认为“列表”）|
| `安静` |最小输出 — 仅 ID 或状态代码 |
| `代理` |带有清理字段的结构化 JSON 信封 — 由 `--json`/`--agent` 自动设置 |

JSON 输出示例：```bash
mem0 search "user preferences" --user-id alice --output json | jq '.data.results[].memory'
```
## 与 AI 代理一起使用

CLI 专为在 AI 代理工具循环内使用而构建。将 `--agent` 或 `--json` 作为全局标志传递到 **any** 命令以激活代理模式：

- 每个命令都会输出一致的 JSON 信封：`{"status", "command", "duration_ms", "scope", "count", "data"}`
- “数据”字段仅包含重要的字段 - ID、内存文本、分数、类别。嘈杂的 API 字段被剥离。
- 所有人类可读的输出都被抑制：没有旋转器，没有颜色，没有横幅。
- 错误以 JSON 形式返回到 stdout，并带有非零退出代码，因此您的代理可以像成功一样捕获它们。```bash
# Drop --agent on any command and get clean, parseable JSON
mem0 --agent search "response preferences" --user-id user-42
mem0 --agent add "User prefers concise responses" --user-id user-42
mem0 --agent list --user-id user-42
mem0 --agent delete --all --user-id user-42 --force
```

  "scope": { "user_id": "alice" },

另外两个代理友好的功能：

- **`--output json`** 返回未经清理的结构化数据 - 当您需要完整的原始 API 响应时非常有用
- **`mem0 help --json`** 以 JSON 形式返回完整的命令树，以便代理可以自行发现可用的命令和选项

对于非交互式环境（CI、代理运行时），通过“mem0 init --api-key m0-xxx --user-id alice --force”或“MEM0_API_KEY”环境变量设置凭据。## 环境变量

|变量|描述 |
|----------|-------------|
| `MEM0_API_KEY` | API 密钥（覆盖配置文件）|
| `MEM0_BASE_URL` | API 基本 URL |
| `MEM0_USER_ID` |默认用户 ID |
| `MEM0_AGENT_ID` |默认代理 ID |
| `MEM0_APP_ID` |默认应用程序 ID |
| `MEM0_RUN_ID` |默认运行 ID |
| `MEM0_ENABLE_GRAPH` |启用图形内存（“true”/“false”）|

环境变量优先于配置文件中的值，配置文件中的值又优先于默认值。

## 全局标志

这些标志可用于所有命令：|旗帜|描述 |
|------|-------------|
| `--json` |启用代理模式：结构化 JSON 信封输出，无颜色或微调器 |
| `--代理` | `--json` 的别名 |
| `--api-key` |覆盖此请求配置的 API 密钥 |
| `--base-url` |覆盖此请求配置的 API 基本 URL |
| `-o, --output` |设置输出格式|
