## Mem0 MCP 是什么？

Mem0 MCP 服务器将 Mem0 的内存功能公开为 MCP 工具，让 AI 代理决定何时保存、搜索或更新信息。云托管的 MCP 服务器无需本地安装 - 只需连接并开始使用内存。

## 快速设置

使用单个命令将 Mem0 MCP 添加到您的首选客户端：```bash
npx mcp-add \
  --name mem0-mcp \
  --type http \
  --url "https://mcp.mem0.ai/mcp" \
  --clients "claude,claude code,cursor,windsurf,vscode,opencode"
```
这会立即自动为所有支持的客户端配置 Mem0 MCP。## 可用工具

MCP 服务器向您的 AI 客户端公开这些内存工具：

|工具|描述 |
|------|-------------|
| `添加内存` |保存用户/代理的文本或对话历史记录 |
| `搜索记忆` |使用过滤器对现有记忆进行语义搜索 |
| `获取记忆` |使用结构化过滤器和分页列出记忆 |
| `获取内存` |通过“memory_id”检索一个内存 |
| `更新内存` |确认ID后覆盖内存文本|
| `删除内存` |通过 `memory_id` 删除单个内存 |
| `删除所有记忆` |批量删除范围内的所有内存|
| `删除实体` |删除用户/代理/应用程序/运行实体及其内存 |
| `列表实体` |枚举存储在 Mem0 | 中的用户/代理/应用程序/运行
| `列表事件` |使用过滤器和分页列出内存操作事件 |
| `获取事件状态` |通过`event_id`检查异步内存操作的状态 |

    **直接 MCP（最快，仅限 MCP）。** Codex 从 `~/.codex/config.toml` 作为 TOML（而非 JSON）读取 MCP 服务器。添加：    ```toml
    [mcp_servers.mem0]
    url = "https://mcp.mem0.ai/mcp"
    bearer_token_env_var = "MEM0_API_KEY"
    ```
在启动 Codex 的 shell 中导出“MEM0_API_KEY”，然后重新启动 Codex。 `codex mcp add` 仅支持 stdio 服务器，因此必须直接通过 `config.toml` 添加 HTTP 服务器，或者通过 Codex 应用程序中的 **插件 → 连接到自定义 MCP → Streamable HTTP** UI 添加。

    <注意>
      Codex 使用服务器名称“mem0”（不像本页上的其他客户端那样使用“mem0-mcp”），因此如果您稍后旁加载它，它会与捆绑插件注册的名称相匹配。

然后运行“codex”和“/plugins”，浏览 **Mem0 插件** 市场，并安装 **Mem0**。不要将其与上面的 Direct MCP 设置结合起来 - 侧面加载插件通过 `.codex-mcp.json` 自动注册 `mem0`，因此手动 `[mcp_servers.mem0]` 块将创建一个副本。

    有关完整详细信息、生命周期挂钩设置和管理命令（“codex 插件市场升级”/“删除”），请参阅 [Codex 集成指南](/integrations/codex)。

## 验证您的设置

配置完成后，您的 AI 客户端可以：
- 使用“add_memory”自动保存信息
- 使用“search_memories”搜索记忆
- 使用“update_memory”更新内存
- 使用“delete_memory”删除内存

**交互示例：**```
User: Remember that I love tiramisu
Agent: Got it! I've saved that you love tiramisu.

User: What do you know about my food preferences?
Agent: Based on your memories, you love tiramisu.

User: Update my project: the mobile app is now 80% complete
Agent: Updated your project status successfully.
```
