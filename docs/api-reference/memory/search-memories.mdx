跨存储记忆的相关性排名混合搜索。 V3 使用多信号检索——语义、BM25 关键字和并行和融合评分的实体匹配。返回的“score”是一个组合的“[0, 1]”值。

实体 ID（“user_id”、“agent_id”、“app_id”、“run_id”）**必须**在“filters”对象内传递 - 顶级实体 ID 将被拒绝，返回 400。至少需要一个实体 ID。

`filters` 对象支持复杂的逻辑运算（AND、OR、NOT）和比较运算符：
- `in`：匹配任何指定的值
- `gte`：大于或等于
- `lte`：小于或等于
- `gt`：大于
- `lt`：小于
- `ne`: 不等于
- `icontains`：不区分大小写的包含检查
- `*`：匹配所有内容的通配符

### 搜索参数默认值

| 参数| V1/V2 | V3 |
| --- | --- | --- |
| `top_k` |支持（默认10） |支持（1-1000，默认10）|
| `阈值` |无默认 |默认 `0.1`（通过 `0.0` 禁用） |
| `重新排名` |默认 `true` |默认“false”（传递“true”以启用）|
