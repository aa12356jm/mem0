> 中文注释：本文件非代码内容已翻译为中文，代码块与配置示例保持原样。










使用 V3 附加管道从对话中提取并存储记忆。端点使用单遍仅添加提取 - 一次 LLM 调用，无更新/删除。记忆随着时间的推移而积累；没有任何内容被覆盖。

## 端点

- **方法**：`POST`
- **网址**：`/v3/memories/add/`
- **内容类型**：`application/json`

处理是异步的。响应返回一个“event_id”，您可以通过“GET /v1/event/{event_id}/”进行轮询。

## 必需的标头

| 标题|必填 |描述 |
| --- | --- | --- |
| `授权：令牌 <MEM0_API_KEY>` |是的 | API 密钥的范围仅限于您的工作区。 |
| `接受：application/json` |是的 |确保 JSON 响应。 |

## 请求正文

为 Mem0 提供对话消息以从中提取记忆。至少需要一个实体 ID（“user_id”、“agent_id”、“app_id”或“run_id”），以便将内存范围限定为会话。实体 ID 在顶层被接受。

<代码组>```json Basic request
{
  "user_id": "alice",
  "messages": [
    { "role": "user", "content": "I moved to Austin last month." }
  ],
  "metadata": {
    "source": "onboarding_form"
  }
}```
</代码组>

### 公共字段

| 领域|类型 |必填 |描述 |
| --- | --- | --- | --- |
| `消息` |数组|是的 |对话转向 Mem0 来提取记忆。每个对象应包含“角色”和“内容”。 |
| `用户 ID` |字符串|否* |将内存与用户相关联。 |
| `agent_id` |字符串|否* |将内存与代理关联起来。 |
| `run_id` |字符串|否* |将内存与运行关联起来。 |
| `app_id` |字符串|否* |将内存与应用程序关联。 |
| `元数据` |对象|可选|自定义键/值元数据（例如，`{"topic": "preferences"}`）。 |
| `推断` |布尔值（默认“true”）|可选|设置为“false”以跳过推理并按原样存储提供的文本。 |

> \* 至少需要一个实体 ID（`user_id`、`agent_id`、`app_id` 或 `run_id`）。

## 回复

该请求排队等待后台处理。响应包含用于跟踪状态的“event_id”。

<代码组>```json 200 response
{
  "message": "Memory processing has been queued for background execution",
  "status": "PENDING",
  "event_id": "evt-uuid"
}
```

```json 400 response
{
  "error": "400 Bad Request",
  "details": {
    "message": "Invalid input data. Please refer to the memory creation documentation at https://docs.mem0.ai/platform/quickstart#4-1-create-memories for correct formatting and required fields."
  }
}
```
</代码组>

<信息>
通过“GET /v1/event/{event_id}/”轮询事件状态。处理完成后，状态将为“SUCCEEDED”或“FAILED”。
</信息>
