本质上，从 LLM 中创建一个同伴就像一个循环一样简单。但这些循环对于一种没有个性化的角色非常有效，但一旦重新启动聊天就会失效。

问题：法学硕士是无国籍的。 GPT 不记得对话。您可以将所有内容填充到上下文窗口中，但这会变得缓慢、昂贵，并且会大规模中断。

解决办法：Mem0.它提取并存储对话中重要的内容，然后在需要时检索它。您的同伴会记住用户偏好、过去的事件和历史。

## 带内存的基本循环

马克斯想要参加马拉松训练。他开始与人工智能跑步教练雷聊天。

<标签>
  <选项卡标题=“平台”>```python
from openai import OpenAI
from mem0 import MemoryClient

openai_client = OpenAI(api_key="your-openai-key")
mem0_client = MemoryClient(api_key="your-mem0-key")

def chat(user_input, user_id):
    # Retrieve relevant memories
    memories = mem0_client.search(user_input, filters={"user_id": user_id}, top_k=5)
    context = "\\n".join(m["memory"] for m in memories["results"])

    # Call LLM with memory context
    response = openai_client.chat.completions.create(
        model="gpt-4o-mini",
        messages=[
            {"role": "system", "content": f"You're Ray, a running coach. Memories:\\n{context}"},
            {"role": "user", "content": user_input}
        ]
    ).choices[0].message.content

    # Store the exchange
    mem0_client.add([
        {"role": "user", "content": user_input},
        {"role": "assistant", "content": response}
    ], user_id=user_id)

    return response
```
</标签>
  <选项卡标题=“开源”>```python
from openai import OpenAI
from mem0 import Memory

OLLAMA_URL = "http://localhost:11434"
CHAT_MODEL = "llama3.1:latest"

memory = Memory.from_config({
    "vector_store": {
        "provider": "qdrant",
        "config": {
            "collection_name": "fitness_companion",
            "host": "localhost",
            "port": 6333,
            "embedding_model_dims": 768,
        },
    },
    "llm": {
        "provider": "ollama",
        "config": {
            "model": CHAT_MODEL,
            "temperature": 0,
            "max_tokens": 2000,
            "ollama_base_url": OLLAMA_URL,
        },
    },
    "embedder": {
        "provider": "ollama",
        "config": {
            "model": "nomic-embed-text:latest",
            "ollama_base_url": OLLAMA_URL,
        },
    },
})

ollama_chat = OpenAI(base_url=f"{OLLAMA_URL}/v1", api_key="ollama")

def chat(user_input, user_id):
    # Retrieve relevant memories
    memories = memory.search(user_input, filters={"user_id": user_id}, top_k=5)
    context = "\n".join(m["memory"] for m in memories["results"])

    # Call LLM with memory context (Ollama via OpenAI-compatible API)
    response = ollama_chat.chat.completions.create(
        model=CHAT_MODEL,
        messages=[
            {"role": "system", "content": f"You're Ray, a running coach. Memories:\n{context}"},
            {"role": "user", "content": user_input},
        ],
    ).choices[0].message.content

    # Store the exchange
    memory.add(
        [
            {"role": "user", "content": user_input},
            {"role": "assistant", "content": response},
        ],
        user_id=user_id,
    )

    return response
```

</标签>

## 按类型组织内存

### 将临时与永久分开

马克斯提到他的膝盖很痛。这与他的马拉松目标不同——一个是暂时的，另一个是长期的。

<标签>
  <选项卡标题=“平台”>
**类别与元数据：**

- **类别**：由 Mem0 根据内容 AI 分配（你不能强迫它们）
- **元数据**：由您手动设置以进行强制标记

在项目级别定义自定义类别。 Mem0 会根据内容自动为内存标记相关类别：```python
mem0_client.project.update(custom_categories=[
    {"goals": "Race targets and training objectives"},
    {"constraints": "Injuries, limitations, recovery needs"},
    {"preferences": "Training style, surfaces, schedules"}
])
```
<注意>
**类别与元数据：** 类别是由 Mem0 根据内容语义 AI 分配的。您定义调色板，Mem0 选择适用的调色板。如果您需要有保证的标记，请改用“元数据”。
</注>

现在，当您添加内存时，Mem0 会自动分配适当的类别：```python
# Add goal - Mem0 automatically tags it as "goals"
mem0_client.add(
    [{"role": "user", "content": "Sub-4 marathon is my A-race"}],
    user_id="max"
)

# Add constraint - Mem0 automatically tags it as "constraints"
mem0_client.add(
    [{"role": "user", "content": "My right knee flares up on downhills"}],
    user_id="max"
)
```
Mem0 读取内容并智能地选择适用的类别。您定义调色板，它处理标记。

**重要提示：** 您不能强制指定特定类别。 Mem0 的平台根据内容决定哪些类别是相关的。如果您需要强制标记某些内容，请使用“元数据”：```python
# Force tag using metadata (not categories)
mem0_client.add(
    [{"role": "user", "content": "Some workout note"}],
    user_id="max",
    metadata={"workout_type": "speed", "forced_tag": "custom_label"}
)
```
</标签>
  <选项卡标题=“开源”>
**通过元数据分类：**

在开源中，模型类别在“元数据”中具有稳定的字段——这里我们使用“memory_bucket”：```python
# Add goal
memory.add(
    [{"role": "user", "content": "Sub-4 marathon is my A-race"}],
    user_id="max",
    metadata={"memory_bucket": "goals"},
)

# Add constraint
memory.add(
    [{"role": "user", "content": "My right knee flares up on downhills"}],
    user_id="max",
    metadata={"memory_bucket": "constraints"},
)
```
<注意>
**类别与元数据：** 在开源中，类别被建模为您在每个“添加”上设置的“元数据”字段。过滤器只能看到您在“add”上添加的内容。
</注>```python

    [{"role": "user", "content": "Some workout note"}],
    user_id="max",
    metadata={"memory_bucket": "goals", "workout_type": "speed", "forced_tag": "custom_label"},
)
```

## 限时记忆

### 自动过期事实

马克斯扭动脚踝。两周后就会痊愈——记忆也会消失。

<标签>
  <选项卡标题=“平台”>```python
from datetime import datetime, timedelta

expiration = (datetime.now() + timedelta(days=14)).strftime("%Y-%m-%d")

mem0_client.add(
    [{"role": "user", "content": "Rolled my left ankle, needs rest"}],
    user_id="max",
    metadata={"memory_bucket": "constraints", "expires_on": expiration}
)
```
将“expires_on”存储在元数据中并定期清理过期的内存。一旦脚踝被移除，雷就不再询问有关脚踝的事情。

## Common Production Patterns

### Episodic Stories with run_id

波士顿的训练与纽约的训练不同。 Separate the memory threads:

<标签>
  <选项卡标题=“平台”>```python

mem0_client.add(messages, user_id="max", run_id="nyc-2025")

# Retrieve only Boston memories
boston_memories = mem0_client.search(
    "training plan",
    user_id="max",
    run_id="boston-2025"
)
```
</标签>
  <选项卡标题=“开源”>```python
memory.add(messages, user_id="max", run_id="boston-2025")
memory.add(messages, user_id="max", run_id="nyc-2025")

</标签>

Each race gets its own episodic boundary.无交叉污染。

### 按日期过滤

优先考虑最近的训练而不是旧的数据：

<标签>
  <选项卡标题=“平台”>```python
recent = mem0_client.search(
    "training progress",
    user_id="max",
    filters={"created_at": {"gte": "2025-10-01"}}
)
```
</标签>
  <选项卡标题=“开源”>```python
# Qdrant range filters require numbers — store an epoch timestamp in metadata
from datetime import datetime

epoch = int(datetime(2025, 10, 15).timestamp())
memory.add(
    [{"role": "user", "content": "Completed 18-mile long run"}],
    user_id="max",
    metadata={"logged_epoch": epoch},
)

cutoff = int(datetime(2025, 10, 1).timestamp())
recent = memory.search(
    "training progress",
    user_id="max",
    filters={"logged_epoch": {"gte": cutoff}},
)
```
</标签>

## 你构建了什么

一个同伴：

- **跨会话持续** - Mem0 存储
- **过滤噪音** - 自定义说明
- **按类型组织** - 类别
- **适应个性** - **`agent_id`**
- **保持快速** - 短期缓冲
- **处理时间事实** - 过期
- **扩展到生产** - 批处理、元数据、修剪

这种模式适用于任何同伴：健身教练、导师、角色扮演角色、治疗机器人、创意写作伙伴。

---

<提示>
从最多 2-3 个类别开始（例如，目标、约束、偏好）。更多类别会降低标记的准确性。在看到 Mem0 提取的内容后，您可以随时添加更多内容。</提示>

## 生产清单

启动前：

- 为您的域设置自定义说明
- 定义 2-3 个类别（目标、约束、偏好）
- 添加有时限事实的过期策略
- 实现 API 调用的错误处理
- 监控内存质量（Mem0 仪表板或本地时的“get_all”/Qdrant）
- 清除生产项目的测试数据
