本指南向您展示如何检索和导出数据以进行检查、迁移或合规性。

## 获取所有记忆

使用带有过滤器的 get_all() 来检索特定用户的所有内容：```python
dev_memories = client.get_all(
    filters={"user_id": "dev"},
    page_size=50
)

print(f"Total memories: {dev_memories['count']}")
print(f"First memory: {dev_memories['results'][0]['memory']}")

```
**输出：**```
Total memories: 1
First memory: Dev works at TechCorp as a senior engineer

```
<信息>
**预期输出：** `get_all()` 检索了 Dev 的完整内存记录。此方法返回与您的过滤器匹配的所有内容 - 没有语义搜索，没有排名，只是原始检索。非常适合出口和审核。
</信息>

您可以按元数据过滤以获取特定类型：```python
carl_projects = client.get_all(
    filters={
        "AND": [
            {"user_id": "carl"},
            {"metadata": {"type": "project"}}
        ]
    }
)

for memory in carl_projects['results']:
    print(memory['memory'])

```
**输出：**```
Carl is leading the API redesign project, targeting Q2 launch

```

## 寻找记忆

当您需要语义搜索而不是检索所有内容时，请使用“search()”：```python
results = client.search(
    query="What does Dev do for work?",
    filters={"user_id": "dev"},
    top_k=5
)

for result in results['results']:
    print(f"{result['memory']} (score: {result['score']:.2f})")

```
**输出：**```
Dev works at TechCorp as a senior engineer (score: 0.89)

```
搜索适用于所有记忆领域，并按相关性排名。当您有特定问题时使用它，当您需要所有内容时使用“get_all()”。

## 导出为结构化格式

为了迁移或合规性，您可以使用 Pydantic 风格的 JSON 模式将内存导出到结构化模式中。

### 第 1 步：定义架构```python
professional_profile_schema = {
    "properties": {
        "full_name": {
            "type": "string",
            "description": "The person's full name"
        },
        "current_role": {
            "type": "string",
            "description": "Current job title or role"
        },
        "company": {
            "type": "string",
            "description": "Current employer"
        }
    },
    "title": "ProfessionalProfile",
    "type": "object"
}

```
### 第 2 步：创建导出作业```python
export_job = client.create_memory_export(
    schema=professional_profile_schema,
    filters={"user_id": "dev"}
)

print(f"Export ID: {export_job['id']}")
print(f"Status: {export_job['status']}")

```
**输出：**```
Export ID: exp_abc123
Status: processing

```
<信息>
**导出已启动：** 状态为“正在处理”。大量导出可能需要几秒钟。在下载数据之前，使用 get_memory_export() 进行轮询，直到状态更改为“已完成”。
</信息>

### 第 3 步：下载导出文件```python
# Get by ID
export_data = client.get_memory_export(
    memory_export_id=export_job['id']
)

print(export_data['data'])

```
**输出：**```json
{
  "full_name": "Dev",
  "current_role": "senior engineer",
  "company": "TechCorp"
}

```
您还可以通过过滤器检索导出：```python
# Get latest export matching filters
export_by_filters = client.get_memory_export(
    filters={"user_id": "dev"}
)

print(export_by_filters['data'])

```

## 添加导出指令

指导 Mem0 如何解决冲突或格式化导出：```python
export_with_instructions = client.create_memory_export(
    schema=professional_profile_schema,
    filters={"user_id": "arjun"},
    export_instructions="""
1. Use the most recent information if there are conflicts
2. Only include confirmed facts, not speculation
3. Return null for missing fields rather than guessing
"""
)

```
<提示>
下载前务必检查导出状态。在循环中短暂延迟调用“get_memory_export()”，直到“status == “completed””。在处理过程中尝试下载会返回不完整的数据。
</提示>

## 平台导出

您还可以直接从 Mem0 平台 UI 导出内存：

1. 导航至项目仪表板中的 **Memory Exports**
2. 单击**创建导出**
3. 选择您的过滤器和架构
4. 下载完成的导出为 JSON

这对于一次性导出或手动数据审核非常有用。

<警告>
导出的数据将在 7 天后过期。如果您需要长期存档，请在本地下载并存储导出内容。过期后，您需要重新创建导出作业。</警告>

---

## 你构建了什么

完整的内存导出系统，多种检索方式：

- **批量检索（get_all）** - 获取所有匹配过滤器的内存以进行全面审核
- **语义搜索** - 基于查询的查找和相关性评分
- **结构化导出** - 用于迁移和合规性的 Pydantic 模式导出
- **导出说明** - 指导冲突解决和数据格式化
- **平台 UI 导出** - 通过仪表板一次性手动下载这涵盖数据可移植性、GDPR 合规性、系统迁移和手动审核。

---

## 总结

使用 **`get_all()`** 进行批量检索，使用 **`search()`** 解决特定问题，使用 **`create_memory_export()`** 进行自定义架构的结构化数据导出。请记住，导出将在 7 天后过期 - 将其下载到本地以进行长期存档。
