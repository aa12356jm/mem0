# Mem0 如何组织内存

Mem0 将内存分层，以便智能体在正确的时间记住正确的细节。把它想象成一个笔记本：当前任务的便签、会话的日记以及用户共享的所有内容的存档。

<信息>
  **为什么重要**
  - 保持对话连贯，无需重复说明。
  - 让客服人员根据长期偏好个性化响应。
  - 通过将内存范围限定到正确的层来避免过度获取数据。
</信息>

## 关键术语

- **对话记忆** – 单回合内的飞行中消息（刚才所说的）。
- **会话内存** – 适用于当前任务或通道的短暂事实。
- **用户记忆** – 与个人、帐户或工作空间相关的长期知识。
- **组织记忆** – 可供多个代理或团队使用的共享上下文。```mermaid

## 短期记忆与长期记忆

短期记忆使当前对话保持连贯。它包括：

- **对话历史记录** – 最近的顺序，以便代理记住刚才所说的内容。
- **工作内存** – 临时状态，例如工具输出或中间计算。
- **注意上下文** – 助理的直接焦点，类似于一个人在句子中间所想到的内容。

长期记忆可以保存不同会话中的知识。它捕获：

- **事实记忆** – 用户偏好、帐户详细信息和领域事实。
- **情景记忆** – 过去互动或已完成任务的总结。
- **语义记忆** – 概念之间的关系，以便代理可以稍后推理它们。

Mem0 将这些经典类别映射到其分层存储上，以便您可以决定哪些内容应该快速消失，哪些内容应该持续数月。

## 它是如何工作的？Mem0 单独存储每一层，并在查询时将它们合并：1. **捕获** – 轮次处于活动状态时，消息进入对话层。
2. **提升** – 相关详细信息根据您的“user_id”、“run_id”和元数据保留到会话或用户内存中。
3. **检索** – 搜索管道从所有层中提取，首先对用户记忆进行排名，然后是会话笔记，然后是原始历史记录。```python
import os

from mem0 import Memory

memory = Memory(api_key=os.environ["MEM0_API_KEY"])

# Sticky note: conversation memory
memory.add(
    ["I'm Alex and I prefer boutique hotels."],
    user_id="alex",
    run_id="trip-planning-2025",
)

# Later in the session, pull long-term + session context
results = memory.search(
    "Any hotel preferences?",
    user_id="alex",
    run_id="trip-planning-2025",
)
```
<提示>
  当您希望短期上下文自动过期时，请使用“run_id”；依靠“user_id”来实现持久的个性化。
</提示>

## 什么时候应该使用每一层？

- **对话记忆** – 仅在当前回合内重要的工具调用或思维链。
- **会话内存** – 完成后应重置的多步骤任务（入门流程、调试会话）。
- **用户记忆** – 在交互过程中必须保留的个人偏好、帐户状态或合规性详细信息。
- **组织记忆** – 每个代理都应该记住的共享常见问题解答、产品目录或政策。## 比较如何

|层|终身|短期还是长期|最适合 |权衡|
| ---| ---| ---| ---| ---|
|对话 |单一回复 |短期|工具执行详情 |回合结束后丢失 |
|会议|分钟到小时 |短期|多步骤流程 |完成后手动清除 |
|用户 |几周到永远|长期|个性化|需要同意/治理 |
|组织|全局配置 |长期|共享知识|需要所有者保持最新状态 |<警告>
  避免将机密或未编辑的 PII 存储在用户或组织内存中 - Mem0 在设计上是可检索的。首先对敏感值进行加密或散列。
</警告>
