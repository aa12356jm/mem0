# 通过更新保持记忆准确

Mem0 的更新操作可让您修复或丰富现有内存，而无需删除它。当用户改变他们的偏好或澄清事实时，使用更新来保持知识库最新。

<信息>
  **为什么重要**
  - 立即纠正过时或不正确的记忆。
  - 添加新的元数据，以便过滤器和重新排序器保持清晰。
  - 适用于一次性编辑和大批量（最多 1000 个内存）。
</信息>

## 关键术语

- **memory_id** – `add` 或 `search` 结果返回的唯一标识符。
- **文本** / **数据** – 替换存储内存值的新内容。
- **元数据** – 您与文本一起更新的可选键值对。
- **timestamp** – Unix 纪元（int/float）或 ISO 8601 字符串来覆盖内存的时间戳。
- **batch_update** – 在单个请求中编辑多个内存的平台 API。- **不可变** – 必须删除并重新添加而不是更新的标记记忆。

## 更新流程如何工作<步骤>
<Step title="找到内存">
使用“搜索”或仪表板检查来捕获要更改的“memory_id”。
</步骤>
<步骤标题=“提交更新”>
使用新文本和可选元数据调用“update”（或“batch_update”）。 Mem0 覆盖存储的值并调整索引。
</步骤>
<步骤标题=“验证”>
检查响应或重新运行“搜索”以确保修改后的内存与新内容一起出现。
</步骤>
</步骤>

## 单次内存更新（平台）

<代码组>```python Python
from mem0 import MemoryClient

client = MemoryClient(api_key="your-api-key")

memory_id = "your_memory_id"
client.update(
    memory_id=memory_id,
    text="Updated memory content about the user",
    metadata={"category": "profile-update"},
    timestamp="2025-01-15T12:00:00Z"
)
```

```javascript JavaScript
import MemoryClient from 'mem0ai';

const client = new MemoryClient({ apiKey: "your-api-key" });
const memory_id = "your_memory_id";

await client.update(memory_id, {
  text: "Updated memory content about the user",
  metadata: { category: "profile-update" },
  timestamp: "2025-01-15T12:00:00Z"
});
```
</代码组>

<信息图标=“检查”>
  预计确认消息和更新的内存几乎会立即出现在仪表板上。
</信息>

## 批量更新（平台）

一次通话最多可更新 1000 个记忆。

<代码组>```python Python
from mem0 import MemoryClient

client = MemoryClient(api_key="your-api-key")

update_memories = [
    {"memory_id": "id1", "text": "Watches football"},
    {"memory_id": "id2", "text": "Likes to travel"}
]

response = client.batch_update(update_memories)
print(response)
```

```javascript JavaScript
import MemoryClient from 'mem0ai';

const client = new MemoryClient({ apiKey: "your-api-key" });

const updateMemories = [
  { memoryId: "id1", text: "Watches football" },
  { memoryId: "id2", text: "Likes to travel" }
];

client.batchUpdate(updateMemories)
  .then(response => console.log('Batch update response:', response))
  .catch(error => console.error(error));
```
</代码组>

## 提示

- 一起更新“文本”**和**“元数据”以保持过滤器准确。
- 批量更新是大量导入后或同步 CRM 更正时的理想选择。
- 不可变的记忆必须被删除并重新添加而不是更新。
- 将更新与反馈信号（赞成/反对）配对，以自动自我修复记忆。

<Callout type="tip" icon="plug">
  **MCP 替代方案**：通过 <Link href="/platform/mem0-mcp">Mem0 MCP</Link>，AI 代理可以在用户更正信息时更新自己的记忆。
</标注>

## 托管与 OSS 的差异

|能力| Mem0平台|内存0 OSS |
| ---| ---| ---|
|更新通话 | `client.update(memory_id, {...})` | `内存.更新（内存_id，数据= ...）` |
|批量更新| `客户。batch_update`（最多 1000 个内存）|编写您自己的循环或批量作业脚本 |
|仪表板可视性 |检查 UI 中的更新 |通过日志或自定义工具检查 |
|不可变的处理 |返回描述性错误 |引发异常——删除并重新添加|
