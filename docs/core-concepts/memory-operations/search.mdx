# Mem0 如何搜索内存

Mem0 的搜索操作让代理可以提出自然语言问题并找回最重要的记忆。就像一个聪明的图书管理员一样，它可以从您存储的所有内容中准确找到您需要的内容。

<信息>
  **为什么重要**
  - 检索正确的事实，无需从头开始重建提示。
  - 支持托管平台和 OSS，因此您可以在本地测试并大规模部署。
  - 保持结果与过滤器、重新排序器和阈值相关。
</信息>

## 关键术语

- **查询** – 您传递给“搜索”的自然语言问题或语句。
- **过滤器** – JSON 逻辑（AND/OR、比较运算符），按用户、类别、日期等缩小结果范围。
- **top_k/threshold** – 控制返回的记忆数量和最小相似度分数。
- **重新排序** – 可选的第二遍，可在配置重新排序器时提高精度。## 架构

## 它是如何工作的？

搜索将您的自然语言问题转换为向量嵌入，然后在数据库中查找具有相似嵌入的记忆。结果按相似度得分进行排名，并且可以通过过滤器或重新排名进一步细化。```python
# Minimal example that shows the concept in action
# Platform API
client.search("What are Alice's hobbies?", filters={"user_id": "alice"})

# OSS
m.search("What are Alice's hobbies?", filters={"user_id": "alice"})
```
<提示>
  始终至少提供一个“user_id”过滤器来将搜索范围限制到正确的用户记忆。这可以防止用户之间的交叉污染。
</提示>

## 什么时候应该使用它？

- **上下文检索** - 当您的代理需要过去的上下文来生成更好的响应时
- **个性化** - 回忆用户偏好、历史记录或过去的交互
- **事实检查** - 在响应之前根据存储的记忆验证信息
- **决策支持** - 当代理需要相关背景信息来做出决策时

## 平台与 OSS 使用情况|能力| Mem0平台|内存0 OSS |
| ---| ---| ---|
| **搜索/get_all 上的实体 ID** |里面 `filters={"user_id": "alice"}` |内部 `filters={"user_id": "alice"}` （与 v3 中的平台一致 — 顶级 kwargs 引发 `ValueError`）|
| **过滤语法** |具有字段级访问的逻辑运算符（“AND”、“OR”、比较）|基本字段过滤器，通过 Python 钩子扩展 |
| **重新排名** |使用托管重新排序器目录切换“rerank=True” |需要配置本地或第三方重新排名器 |
| **阈值** |请求级配置（`threshold`、`top_k`）|通过SDK参数控制|
| **响应元数据** |包括置信度分数、时间戳、仪表板可见性 |由您的存储后端决定 |

## 使用 Mem0 平台搜索

<代码组>```python Python
from mem0 import MemoryClient

client = MemoryClient(api_key="your-api-key")

query = "What do you know about me?"
filters = {
   "OR": [
      {"user_id": "alice"},
      {"agent_id": {"in": ["travel-assistant", "customer-support"]}}
   ]
}

results = client.search(query, filters=filters)
```

```javascript JavaScript
import { MemoryClient } from "mem0ai";

const client = new MemoryClient({apiKey: "your-api-key"});

const query = "I'm craving some pizza. Any recommendations?";
const filters = {
  AND: [
    { user_id: "alice" }
  ]
};

const results = await client.search(query, {
  filters
});
```
</代码组>

## 使用 Mem0 开源搜索

<代码组>```python Python
from mem0 import Memory

m = Memory()

# Simple search — entity IDs go in `filters`
related_memories = m.search("Should I drink coffee or tea?", filters={"user_id": "alice"})

# Search with additional metadata filters (combine entity + metadata in the same dict)
memories = m.search(
    "food preferences",
    filters={"user_id": "alice", "categories": {"contains": "diet"}},
)
```

```javascript JavaScript
import { Memory } from 'mem0ai/oss';

const memory = new Memory();

// Simple search — entity IDs go inside `filters`
const relatedMemories = memory.search("Should I drink coffee or tea?", {
    filters: { userId: "alice" },
});

// Combine entity + metadata filters in the same filters object
const memories = memory.search("food preferences", {
    filters: { userId: "alice", categories: { contains: "diet" } },
});
```
</代码组>

</信息>

## 过滤模式

过滤器有助于缩小搜索结果范围。常见用例：

**按会话上下文过滤：**

*平台API：*```python
# Get memories from a specific agent session
client.search("query", filters={
    "AND": [
        {"user_id": "alice"},
        {"agent_id": "chatbot"},
        {"run_id": "session-123"}
    ]
})
```
*操作系统：*```python
# Get memories from a specific agent session — entity IDs combined in filters
m.search("query", filters={
    "user_id": "alice",
    "agent_id": "chatbot",
    "run_id": "session-123",
})
```
**按日期范围过滤：**```python
# Platform only - date filtering
client.search("recent memories", filters={
    "AND": [
        {"user_id": "alice"},
        {"created_at": {"gte": "2024-07-01"}}
    ]
})
```
**按类别过滤：**```python
# Platform only - category filtering
client.search("preferences", filters={
    "AND": [
        {"user_id": "alice"},
        {"categories": {"contains": "food"}}
    ]
})
```
## 更好的搜索技巧

- **使用自然语言**：Mem0 理解意图，因此自然地描述您正在寻找的内容
- **带有用户 ID 的范围**：始终提供 `user_id` 以将搜索范围搜索到相关内存
  - **平台 API**：使用 `filters={"user_id": "alice"}`
  - **OSS**：使用 `user_id="alice"` 作为参数
- **组合过滤器**：使用 AND/OR 逻辑创建精确查询（平台）
- **考虑通配符过滤器**：使用通配符过滤器（例如`run_id：“*”`）进行更广泛的匹配
- **调整参数**：调整结果计数的“top_k”，相关性截止的“threshold”
- **启用重新排名**：当您配置了重新排名器时，使用“rerank=True”（默认）
