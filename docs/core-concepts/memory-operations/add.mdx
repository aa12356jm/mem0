# Mem0 如何添加内存

添加内存是 Mem0 从对话中捕获有用详细信息的方式，以便您的代理稍后可以重复使用它们。可以将其视为将聊天记录中的重要句子保存到代理可以搜索的结构化笔记本中。

<信息>
  **为什么重要**
  - 跨会话保留用户偏好、目标和反馈。
  - 支持下游对话中的个性化和决策。
  - 保持托管平台和 OSS 部署之间的上下文一致。
</信息>

## 关键术语

- **消息** – 用户/助理的有序列表将您发送到“添加”。
- **Infer** – 控制 Mem0 是否提取结构化内存（默认为“infer=True”）或存储原始消息。
- **元数据** – 可选过滤器（例如，`{"category": "movie_recommendations"}`）可改善以后的检索。- **用户/会话标识符** – `user_id`、`agent_id` 或 `run_id` 确定内存范围以供将来搜索。

## 它是如何工作的？

Mem0 提供两种流程：

- **Mem0 平台** – 具有仪表板、缩放和图形功能的完全托管 API。
- **Mem0 开源** – 您在自己的环境中运行的本地 SDK。两个流采用相同的有效负载并通过相同的管道传递。

<步骤>
<步骤title="信息提取">
Mem0 通过 LLM 发送消息，提取需要记住的关键事实、决定或偏好。
</步骤>
<步骤标题=“冲突解决”>
检查现有记忆是否有重复或矛盾，以便最新的事实获胜。
</步骤>
<步骤标题=“存储”>
生成的内存存储在托管矢量存储（和可选的图形存储）中，因此将来的搜索可以快速返回它们。
</步骤>
</步骤>

<警告>
仅当您让 Mem0 推断内存（“infer=True”，默认值）时，重复保护才会在冲突解决步骤期间运行。如果您切换到“infer=False”，Mem0 会完全按照提供的方式存储您的有效负载，因此重复项将会出现。对于同一事实混合使用两种模式将节省两次。
</警告>

您可以通过单个“add”调用触发此管道，无需手动编排。

## 添加Mem0平台

<代码组>```python Python
from mem0 import MemoryClient

client = MemoryClient(api_key="your-api-key")

messages = [
    {"role": "user", "content": "I'm planning a trip to Tokyo next month."},
    {"role": "assistant", "content": "Great! I’ll remember that for future suggestions."}
]

client.add(
    messages=messages,
    user_id="alice",
)
```

```javascript JavaScript
import { MemoryClient } from "mem0ai";

const client = new MemoryClient({apiKey: "your-api-key"});

const messages = [
  { role: "user", content: "I'm planning a trip to Tokyo next month." },
  { role: "assistant", content: "Great! I’ll remember that for future suggestions." }
];

await client.add(messages, {
  user_id: "alice",
});
```
</代码组>

<信息图标=“检查”>
  响应中应包含“memory_id”（或 ID 列表）。检查 Mem0 仪表板以确认正确用户下的新条目。
</信息>

## 使用 Mem0 开源添加

<代码组>```python Python
import os
from mem0 import Memory

os.environ["OPENAI_API_KEY"] = "your-api-key"

m = Memory()

messages = [
    {"role": "user", "content": "I'm planning to watch a movie tonight. Any recommendations?"},
    {"role": "assistant", "content": "How about thriller movies? They can be quite engaging."},
    {"role": "user", "content": "I'm not a big fan of thriller movies but I love sci-fi movies."},
    {"role": "assistant", "content": "Got it! I'll avoid thriller recommendations and suggest sci-fi movies in the future."}
]

# Store inferred memories (default behavior)
result = m.add(messages, user_id="alice", metadata={"category": "movie_recommendations"})

# Optionally store raw messages without inference
result = m.add(messages, user_id="alice", metadata={"category": "movie_recommendations"}, infer=False)
```

```javascript JavaScript
import { Memory } from 'mem0ai/oss';

const memory = new Memory();

const messages = [
  {
    role: "user",
    content: "I like to drink coffee in the morning and go for a walk"
  }
];

const result = memory.add(messages, {
  userId: "alice",
  metadata: { category: "preferences" }
});
```
</代码组>

<提示>
  仅当您需要存储原始成绩单时才使用“infer=False”。大多数工作流程都受益于 Mem0 自动提取结构化内存。
</提示>

<警告>
如果您确实选择“infer=False”，请保持一致。原始插入会跳过冲突解决，因此稍后使用相同内容的“infer=True”调用将创建第二个内存，而不是更新第一个内存。
</警告>

## 什么时候应该添加内存？

每当你的代理学到一些有用的东西时就添加记忆：

- 共享新的用户偏好
- 做出决定或建议
- 目标或任务已完成
- 引入一个新实体
- 用户提供反馈或澄清

存储此上下文可以让智能体在未来的交互中更好地进行推理。
