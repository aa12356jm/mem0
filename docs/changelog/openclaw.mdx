<更新标签=“2026-04-29”描述=“v1.0.11”>**新功能：**
- **技能模式自动设置：** `enableSkillsConfig()` 现在在入职后自动运行 — 使用 `tools.profile = "full"` 启用分类、召回（通过重新排名 + 关键字搜索）和梦想整合，并禁用内置会话内存挂钩以避免冲突
- **内存运行时功能：**插件现在在注册的内存功能上公开 `runtime.getMemorySearchManager()` 和 `resolveMemoryBackendConfig()`，使 OpenClaw 网关能够直接查询内存状态和后端配置
- **维度感知集合：** OSS 向导检测嵌入器维度变化并自动创建一个新集合 (`mem0_<dims>d`)，并警告旧内存在新嵌入器下无法访问
- **技能中的工具文档：** “记忆分类”和“记忆梦想”技能。md 文件现在包含完整的工具参考部分，列出了所有可用工具和参数**改进：**
- **自动捕获和自动调用默认启用：** `autoCapture` 和 `autoRecall` 现在默认为“true”（原为“false”）。清单描述已相应更新。在技能模式下被忽略
- **`memory_update` 优于删除+添加：** 现在，技能更喜欢使用 `memory_update` 进行就地编辑 — 原子性并保留编辑历史记录。更新整合模式：更新最佳内存，删除冗余内存
- **搜索阈值降低：** 默认“searchThreshold”从“0.5”降低到“0.1”，以实现更广泛的召回。删除了硬编码的“0.6”召回特定覆盖 - 所有搜索现在都使用配置的阈值
- **嵌入器尺寸传播：** 如果未明确设置，矢量存储配置会自动解析嵌入器配置中的尺寸。同步“dimension”和“embeddingModelDims”字段以实现 Qdrant/PGVector 兼容性
- **配置文件写入安全性：** `writeFullConfig()` 现在会在写入之前重新读取并深度合并 `plugins` 部分，保留 OpenClaw 网关写入的 `installs` 和 `slots`
- **其他嵌入器模型：** 在已知嵌入器尺寸中添加了 `mxbai-embed-large` (1024)、`all-minilm` (384) 和 `snowflake-arctic-embed` (1024)**安全：**
- 通过 pnpm 覆盖将 `protobufjs` 提升到 `>=7.5.5` (GHSA-xq3m-2v4x-88gg) ([#5012](https://github.com/mem0ai/mem0/pull/5012))**修复：**
- 移动了“bootstrapTelemetryFlag()”并从模块级副作用中删除了“ensureInstallRecord()”——两者现在都在“register()”内部运行，以避免在 OpenClaw 网关外部加载时发生崩溃
- 修复了 OSS 历史数据库路径解析：绝对路径不再通过 `resolvePath()` 传递，防止双前缀错误
- 使用“id”+“envVars”将清单“providerAuthEnvVars”替换为符合规范的“setup.providers”格式

**依赖关系：**
- 将“mem0ai”从“3.0.1”提升到“3.0.2”
- 将 `pluginApi` 和 `minGatewayVersion` 兼容性提高到 `>=2026.4.24`
