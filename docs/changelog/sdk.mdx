<更新标签=“2026-04-14”描述=“v2.0.0”>**主要版本** — 具有 V3 内存管道、仅 ADD 提取和清理的 API 表面的 Python SDK。**新功能：**
- **单通道提取：** 使用“ADDITIVE_EXTRACTION_PROMPT”将 2-LLM 调用管道替换为附加提取。内存通过“linked_memory_ids”累积 - 不再有更新/删除事件（[#4805](https://github.com/mem0ai/mem0/pull/4805)）
- **混合搜索：** 组合语义 + BM25 关键字匹配 + 具有附加评分的实体增强。原生 `keyword_search()` 添加到 15 个矢量存储适配器（Qdrant、Elasticsearch、OpenSearch、Azure AI Search、Weaviate、Redis、PGVector、Pinecone、Databricks、MongoDB、Milvus、百度、Upstash、Azure MySQL、Vertex AI）（[#4805](https://github.com/mem0ai/mem0/pull/4805)）
- **实体提取和链接：** 基于 spaCy 的实体提取，使用第二个向量集合（`{collection}_entities`）进行跨内存关系检索。可选依赖项：`pip install mem0ai[nlp]` ([#4805](https://github.com/pipinstallmem0ai[nlp])com/mem0ai/mem0/pull/4805))
- **批量操作：** 同步“Memory”和异步“AsyncMemory”的批量嵌入、批量持久和批量实体链接（8 阶段管道）（[#4805](https://github.com/mem0ai/mem0/pull/4805)）
- **消息持久化：** LLM 上下文的基于 SQLite 的滚动窗口（每个会话范围 10 条消息）（[#4805](https://github.com/mem0ai/mem0/pull/4805)）
- **Valkey 集群模式：** 为启用 Valkey 集群模式 (CME) 的部署添加了 `cluster_mode` 参数 ([#4759](https://github.com/mem0ai/mem0/pull/4759))
- **V3 API 端点：** `MemoryClient.add()` 现在发布到 `/v3/memories/add/`； `MemoryClient.get_all()` 发送到 `/v3/memories/` 并返回一个分页信封 `{"count": int, "next": str |无，“上一个”：str |无，“结果”：[...]}` ([#4856](https://github.com)com/mem0ai/mem0/pull/4856))
- **默认模型：** `gpt-5-mini` 现在是 `OpenAILLM`、`OpenAIStructuredLLM`、`AzureOpenAILLM`、`AzureOpenAIStructuredLLM` 和 `LiteLLM` 后备的默认模型 ([#4829](https://github.com/mem0ai/mem0/pull/4829))**重大变更：**
- **`add()` 返回仅添加事件** — 不再有“UPDATE”或“DELETE”事件。记忆不断积累；没有任何内容被覆盖（[#4805](https://github.com/mem0ai/mem0/pull/4805)）
- **`search()` 默认 `threshold` 现在为 `0.1`** — 为之前的行为传递 `threshold=0.0` ([#4805](https://github.com/mem0ai/mem0/pull/4805))
- **`search()` `score` 现在是一种组合的多信号分数** — 顶级 `score` 将语义相似性、BM25 关键字匹配和实体增强融合为一个值。绝对数字相对于旧的原始余弦分数发生变化；针对代表性查询重新调整任何硬阈值。每个信号的分数不会在响应中公开（[#4805](https://github.com/mem0ai/mem0/pull/4805)、[#4836](https://github.com/mem0ai/mem0/pull/4836))
- **`search()` 默认 `rerank` 现在为 `False`** — 为之前的行为传递 `rerank=True` ([#4805](https://github.com/rerank)com/mem0ai/mem0/pull/4805))
- **`Memory.get_all()` 和 `Memory.search()` 中`top_k` 默认值更改为 100 → 20**（同步 + 异步）。显式传递 `top_k=100` 以恢复旧行为 ([#4843](https://github.com/mem0ai/mem0/pull/4843))
- **实体 ID 验证：** `user_id` / `agent_id` / `run_id` 被修剪；空字符串和仅空白值现在引发 `ValueError` ([#4843](https://github.com/mem0ai/mem0/pull/4843))
- **搜索参数验证：** `threshold` 必须是 `[0, 1]` 中的数字； `top_k` 必须是非负整数 - 无效输入引发 `ValueError` ([#4843](https://github.com/mem0ai/mem0/pull/4843))
- **Memory.add()`中的`messages`拒绝无效类型：**传递`None`或非`(str | dict | list)`值会引发`Mem0ValidationError` (`error_code="VALIDATION_003"`) ([#4843](https://github.com/mem0ai/mem0/pull/4843))
- **`qdrant-client>=1.12。需要 0`** — 从 `>=1.9.1` 升级 ([#4805](https://github.com/mem0ai/mem0/pull/4805))
- **`org_id` 和 `project_id` 已删除** — 从 `MemoryClient` 构造函数和所有方法签名中删除 ([#4740](https://github.com/mem0ai/mem0/pull/4740))
- **删除图形内存 (OSS)：** `mem0/memory/graph_memory.py`、`memgraph_memory.py`、`kuzu_memory.py`、`apache_age_memory.py` 和 `mem0/graphs/`（Neo4j / Memgraph / Kuzu / Apache AGE / Neptune 驱动程序）已删除 — 约 4,000 行。 OSS SDK不再支持图内存；可以卸载图形驱动程序（neo4j、memgraph、kuzu 等）。使用平台 API 来实现图形功能。从配置中删除 `enable_graph` 和 `graph_store` ([#4805](https://github.com/mem0ai/mem0/pull/4805))
- **`enable_graph` 从客户端 SDK 中删除** — 图内存现在是平台上的项目级设置。从“MemoryClient”中删除“enable_graph”。add()` / `search()` / `get_all()` / `update_project()` 调用 ([#4776](https://github.com/mem0ai/mem0/pull/4776))
