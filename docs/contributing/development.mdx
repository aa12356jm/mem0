# 开发贡献

我们努力做出贡献**轻松、协作且愉快**。请按照以下步骤操作，以确保贡献过程顺利进行。

## 通过 PR 提交您的贡献

要做出贡献，请按照以下步骤操作：

1. **分叉并克隆**存储库：[GitHub 上的 Mem0](https://github.com/mem0ai/mem0)
2. **创建功能分支**：使用专用分支进行更改，例如“feature/my-new-feature”
3. **实施更改**：如果添加功能或修复错误，请确保：
   - 编写必要的**测试**
   - 添加**文档、文档字符串和可运行示例**
4. **代码质量检查**：
   - 运行 **linting** 来捕获样式问题
   - 确保**所有测试通过**
5. **提交拉取请求**

有关拉取请求的详细指南，请参阅 [GitHub 的文档](https://docs.github.com)。com/en/pull-requests/collaborating-with-pull-requests/propose-changes-to-your-work-with-pull-requests/creating-a-pull-request)。

## 依赖管理

我们使用“hatch”作为包管理器。按照[官方说明](https://hatch.py​​pa.io/latest/install/)安装它。**不要使用 `pip` 或 `conda` 进行依赖管理。** 相反，请按顺序执行以下步骤：```bash
# 1. Install base dependencies
make install

# 2. Activate virtual environment (this will install dependencies)
hatch shell  # For default environment
hatch -e dev_py_3_11 shell  # For dev_py_3_11 (differences are mentioned in pyproject.toml)

# 3. Install all optional dependencies
make install_all
```

## 开发标准

### 预提交挂钩

确保在贡献之前安装了“pre-commit”：```bash
pre-commit install
```
### 使用 `ruff` 进行 Linting

在提交 PR 之前运行 linter 并修复任何报告的问题：```bash
make lint
```
### 代码格式化

要保持一致的代码风格，请格式化您的代码：```bash
make format
```
### 使用 `pytest` 进行测试

在提交 PR 之前运行测试以验证功能：```bash
make test
```
**注意：** 一些依赖项已从主要依赖项中删除，以减少包大小。在运行测试之前运行“make install_all”来安装必要的依赖项。

目前，发布是手动处理的。我们的目标是频繁发布，通常是在引入新功能或错误修复时。
