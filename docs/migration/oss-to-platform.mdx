# 从开源迁移到平台

将您的 Mem0 实施转移到具有企业功能的托管基础设施。

|范围 |努力|停机时间 |
| -------------------- | -------------- | ---------------------------- |
|基础设施和代码 |低（约 30 分钟）|无（可以并行运行）|

<信息>
  **为什么迁移到平台？**

  - **上市时间**：5 分钟内即可完成设置，而 OSS 配置则需要 30 多分钟
  - **企业就绪**：SOC2 Type II 合规性、GDPR 支持、审核日志
  - **高级功能**：Webhooks、内存导出、分析仪表板、自定义类别
  - **多租户**：开箱即用的组织、项目和团队管理
  - **零基础设施**：无矢量数据库、LLM 提供商或维护费用
  - **增强搜索**：重新排名、关键字扩展和高级过滤器
  - **生产级**：自动扩展、高可用性、专门支持
</信息>

## 计划1. **注册**：在 <a href="https://app.mem0.ai?utm_source=oss&utm_medium=migration-oss-to-platform" rel="nofollow">Mem0 平台</a>上创建一个帐户。
2. **获取 API 密钥**：导航至 **设置 > API 密钥** 并生成新密钥。
3. **查看用法**：确定在何处实例化“Memory”以及在何处调用“search”或“get_all”。

## 迁移

### 1.安装或更新SDK

确保您拥有最新版本的 SDK，它同时支持 OSS 和平台客户端。```bash
pip install mem0ai --upgrade
```
### 2.更新初始化

从本地“Memory”类切换到托管“MemoryClient”。```python Open Source (Old)
from mem0 import Memory

config = {
    "vector_store": {
        "provider": "qdrant",
        "config": {"host": "localhost", "port": 6333}
    },
    "llm": {
        "provider": "openai",
        "config": {"model": "gpt-4"}
    }
}

m = Memory.from_config(config)
```

```python Platform (New)
from mem0 import MemoryClient
import os

# Set MEM0_API_KEY in environment or pass explicitly
client = MemoryClient(api_key="m0-...")
```
<信息图标=“检查”>
  运行 `client.get_all(filters={"user_id": "test_connection"})` 来验证您的 API 密钥是否有效。它应该返回一个空列表或有效结果。
</信息>

### 3. 更新检索调用（关键）

<警告>
  **关键更改**：平台使用 v2 端点，要求将过滤参数嵌套在“filters”字典中。
</警告>

<注意>
  所有 SDK 中的“limit”参数已被删除，取而代之的是“top_k”。使用“limit=”更新任何代码以使用“top_k=”。
</注>

|方法|开源 |平台|
| ------ | ----------- | -------- |
| `搜索()` | `m.search(query, user_id="alex")` | `client.search(query,filters={"user_id": "alex"})` |
| `get_all()` | `m.get_all(user_id="alex")` | `client.get_all(filters={"user_id": "alex"})` |
| `添加()` | `m.add(内存, user_id="alex")` | `client.add(内存, user_id="alex")` |
| `删除()` | `米。删除（内存_id）` | `client.delete(memory_id)` |
| `delete_all()` | `m.delete_all(user_id="alex")` | `client.delete_all(user_id="alex")` |

注意：`add()` 和 `delete()` 方法保持不变。 `update()` 方法在平台中不可用 - 请改用删除 + 添加模式。<手风琴组>

## 变更摘要

|特色 |开源 |平台|需要采取行动|
| -------- | ----------- | -------- | ---------------- |
| **初始化** | `Memory.from_config(config)` | `MemoryClient(api_key)` |用 API 密钥替换配置对象 |
| **搜索方式** | `m.search(query, user_id="x")` | `client.search(query,filters={"user_id": "x"})` |将过滤参数移至“filters”字典中 |
| **获取全部方法** | `m.get_all(user_id="x")` | `client.get_all(filters={"user_id": "x"})` |将过滤参数移至“filters”字典中 |
| **添加方法** | `m.add(内存, user_id="x")` | `client.add(内存, user_id="x")` |没有变化|
| **删除方法** | `m.delete(memory_id)` | `client.delete(memory_id)` |没有变化|
| **全部删除** | `m.delete_all(user_id="x")` | `client.delete_all(user_id="x")` |没有变化|
| **更新方法** | `米。更新（内存 ID，新内存）` |使用删除+添加模式 |替换为删除然后添加 |
| **配置** |本地向量存储 + LLM 配置 |托管云基础设施|删除本地配置设置 |

## 回滚计划

如果遇到问题，您可以通过切换回导入来立即恢复。1. **恢复代码**：将`MemoryClient`更改回`Memory`。
2. **恢复配置**：取消注释您的本地矢量存储和 LLM 配置。
3. **验证**：确保您的本地矢量数据库仍在运行且可访问。
