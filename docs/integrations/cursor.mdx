使用 Mem0 插件将持久内存添加到 [**Cursor**](https://cursor.com)。您的 AI 助手会忘记会话之间的所有内容 - 该插件通过 MCP 连接到 Mem0 的云存储层，自动捕获关键生命周期点的学习内容，并在每次响应之前检索相关上下文，从而解决了这个问题。

## 概述

1. **MCP服务器** — 连接Mem0的远程MCP服务器以获取内存工具（添加、搜索、更新、删除）
2. **Lifecycle Hooks** - 在会话启动、压缩和用户提示时自动捕获内存（市场安装）
3. **SDK 技能** — 教代理如何将 Mem0 SDK 集成到您的应用程序中
4. **零本地依赖** — 云托管的 MCP 服务器，无需本地设置

## 先决条件

在使用 Cursor 设置 Mem0 之前，请确保您拥有：

1. Mem0 平台帐户和 API 密钥：
   - <a href="https://app.mem0.ai?utm_source=oss&utm_medium=integration-cursor" rel="nofollow">在 app.mem0.ai 注册</a>
   - <a href="https://app.mem0.ai/dashboard/api-keys?utm_source=oss&utm_medium=integration-cursor" rel="nofollow">获取您的 API 密钥</a>（以 `m0-` 开头）

2.已安装光标（[cursor.com](https://cursor.com)）3. 在 shell 中导出您的 API 密钥：```bash
export MEM0_API_KEY="m0-your-api-key"
```
<警告>
  已在 Cursor 中将 `mem0` 配置为 MCP 服务器？安装前从光标 MCP 设置中删除现有条目，以避免重复工具。
</警告>

### 选项 B — npx（仅限 MCP）```bash
npx mcp-add \
  --name mem0-mcp \
  --type http \
  --url "https://mcp.mem0.ai/mcp" \
  --clients "cursor"
```
### 选项 C — 手动配置（仅限 MCP）

将以下内容添加到“.cursor/mcp.json”：```json
{
  "mcpServers": {
    "mem0": {
      "url": "https://mcp.mem0.ai/mcp/",
      "headers": {
        "Authorization": "Token ${env:MEM0_API_KEY}"
      }
    }
  }
}
```

## 可用的 MCP 工具

安装后，每个 Cursor 会话中都可以使用以下工具：|工具|描述 |
|------|-------------|
| `添加内存` |保存用户/代理的文本或对话历史记录 |
| `搜索记忆` |使用过滤器跨记忆进行语义搜索 |
| `获取记忆` |使用过滤器和分页列出记忆 |
| `获取内存` |通过ID检索特定内存|
| `更新内存` |按 ID 覆盖内存文本 |
| `删除内存` |通过ID删除单个内存|
| `删除所有记忆` |批量删除范围内的所有内存|
| `删除实体` |删除用户/代理/应用程序/运行实体及其内存 |
| `列表实体` |列出存储在 Mem0 中的用户/代理/应用程序/运行 |

## 生命周期挂钩（市场安装）

通过 Cursor Marketplace 安装时，Mem0 会挂钩 Cursor 的生命周期：

### 会话开始
在每个新会话中，插件都会提示代理调用“search_memories”以加载先前会话中的相关上下文。### 用户提示
在处理每个用户消息之前，插件会在 Mem0 中搜索相关记忆并将其注入上下文中。跳过简短的提示以最大限度地减少延迟。

### 预压实
在上下文压缩之前，该插件会捕获全面的会话摘要，因此当上下文窗口重置时不会丢失任何内容。## 工作流程示例```text
# Session 1: Debugging a performance issue
You: The API endpoint /users is taking 3 seconds. Help me optimize it.

# Cursor agent searches memories, proceeds with investigation.
# After completing the task, Mem0 stores:
#   - Learning: "N+1 query in UserService.getAll() — fixed with eager loading"
#   - Decision: "Added database index on users.email column"
#   - Preference: "User prefers query-level fixes over caching"

# Session 2 (next week): Similar issue
You: The /orders endpoint is also slow, same pattern as before.

# Agent searches memories, retrieves the optimization learnings.
# Immediately checks for N+1 queries and missing indexes.
```

- **没有出现工具** — 转到光标设置 > MCP 并验证“mem0”服务器显示为已连接
