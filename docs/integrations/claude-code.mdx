使用 Mem0 插件将持久内存添加到 [**Claude Code**](https://docs.anthropic.com/en/docs/claude-code) (CLI) 和 **Claude Cowork**（桌面应用程序）。您的代理会忘记会话之间的所有内容 - 该插件通过 MCP 连接到 Mem0 的云存储层、自动捕获关键生命周期点的学习内容并在每次响应之前检索相关上下文来修复该问题。

## 概述

1. **MCP服务器** — 连接Mem0的远程MCP服务器以获取内存工具（添加、搜索、更新、删除）
2. **生命周期挂钩** - 在会话开始、上下文压缩、任务完成和会话结束时自动捕获内存
3. **SDK 技能** — 教代理如何将 Mem0 SDK 集成到您的应用程序中
4. **零本地依赖** — 云托管的 MCP 服务器，无需本地设置

## 先决条件

在使用 Claude Code 设置 Mem0 之前，请确保您拥有：1. Mem0 平台帐户和 API 密钥：
   - <a href="https://app.mem0.ai?utm_source=oss&utm_medium=integration-claude-code" rel="nofollow">在 app.mem0.ai 注册</a>
   - <a href="https://app.mem0.ai/dashboard/api-keys?utm_source=oss&utm_medium=integration-claude-code" rel="nofollow">获取您的 API 密钥</a>（以 `m0-` 开头）

2. 安装 Claude Code CLI 或 Claude Cowork 桌面应用程序

3. 在 shell 中导出您的 API 密钥：```bash
export MEM0_API_KEY="m0-your-api-key"
```
## 安装

### 选项 A — 插件市场（推荐）

安装完整的插件，包括 MCP 服务器、生命周期挂钩和 SDK 技能。

1. 添加Mem0市场：   ```
   /plugin marketplace add mem0ai/mem0
   ```
2.安装插件：   ```
   /plugin install mem0@mem0-plugins
   ```
**Claude Cowork 桌面应用程序：** 打开 Cowork 选项卡，单击侧边栏中的 **自定义**，单击 **浏览插件**，然后安装 Mem0。

### 选项 B — 仅 MCP

使用单个命令直接添加 Mem0 MCP 服务器：```bash
npx mcp-add \
  --name mem0-mcp \
  --type http \
  --url "https://mcp.mem0.ai/mcp" \
  --clients "claude code"
```
这为您提供了 MCP 工具，但没有生命周期挂钩或 SDK 技能。

### 选项 C — 手动 MCP 配置

添加到您的 Claude Code MCP 配置（`.mcp.json`）：```json
{
  "mcpServers": {
    "mem0": {
      "type": "http",
      "url": "https://mcp.mem0.ai/mcp/",
      "headers": {
        "Authorization": "Token ${MEM0_API_KEY}"
      }
    }
  }
}
```

## 可用的 MCP 工具

安装后，每个 Claude Code 会话中都可以使用以下工具：

|工具|描述 |
|------|-------------|
| `添加内存` |保存用户/代理的文本或对话历史记录 |
| `搜索记忆` |使用过滤器跨记忆进行语义搜索 |
| `获取记忆` |使用过滤器和分页列出记忆 |
| `获取内存` |通过ID检索特定内存|
| `更新内存` |按 ID 覆盖内存文本 |
| `删除内存` |通过ID删除单个内存|
| `删除所有记忆` |批量删除范围内的所有内存|
| `删除实体` |删除用户/代理/应用程序/运行实体及其内存 |
| `列表实体` |列出存储在 Mem0 中的用户/代理/应用程序/运行 |

## 生命周期挂钩当通过插件市场安装时，Mem0 会挂钩 Claude Code 的生命周期来自动管理内存：

### 会话开始
在每个新会话中，该插件都会提示 Claude 调用“search_memories”以加载先前会话中的相关上下文。在恢复或压缩后会话中，它会相应地调整提示。

### 用户提示
在处理每个用户消息之前，插件会在 Mem0 中搜索与当前提示相关的内存，并将它们注入上下文中。跳过短提示（< 20 个字符）以最大限度地减少延迟。

### 预压实
在上下文压缩之前，该插件会提示 Claude 存储全面的会话摘要 - 包括目标、成就、决策、修改的文件和当前状态 - 因此不会丢失任何内容。### 任务完成
每次任务完成后，该插件都会提示 Claude 提取并存储关键知识：成功的策略、失败的方法、架构决策和新约定。

### 会议结束
当 Claude 完成响应时，该插件会提示任何未存储的学习内容，并通过 Mem0 REST API 捕获转录状态作为后台安全网。

## 工作流程示例```text
# Session 1: Working on a feature
You: Let's refactor the auth module to use JWT tokens instead of sessions.

# Claude searches memories, finds nothing relevant, proceeds with the work.
# After completing the task, Mem0 stores:
#   - Decision: "Migrated auth from sessions to JWT tokens"
#   - Files modified: auth/middleware.ts, auth/token.ts
#   - User preference: "Prefers TypeScript, uses ESLint"

# Session 2 (days later): Related work
You: Add refresh token rotation to the auth system.

# Claude searches memories, retrieves the JWT migration context.
# Knows the file structure, decisions made, and user preferences.
# Continues seamlessly without re-explaining the codebase.
```
