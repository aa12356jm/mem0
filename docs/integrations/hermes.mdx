为 [Hermes Agent](https://github.com/NousResearch/hermes-agent) 添加长期记忆——Nous Research 的一个自我改进的 AI 代理 CLI。 Hermes 有一个可插拔内存系统，Mem0 是支持的提供者之一。启用后，Mem0 会自动从您的对话中学习事实，并在每次对话前显示相关事实 - 所有这些都不会减慢聊天速度。

## 概述

Hermes 与一个外部提供商一起运行一个内置内存系统（基于文件的“MEMORY.md”和“USER.md”）。当 Mem0 处于活动状态时，它会在每个对话回合的三个关键时刻与内置系统相加工作：

### 1. 在代理响应之前（预取）

当你发送消息时，Hermes 会检查是否已经缓存了上一轮的 Mem0 搜索结果。如果是这样，这些记忆将被注入系统提示符中，以便法学硕士可以看到它们。这是**零延迟**——无需等待 API 调用。### 2. 代理响应后（同步）一旦 LLM 完成响应，Hermes 就会在 **后台线程** 中将“（用户消息，助理响应）”对发送到 Mem0 的 API。 Mem0 的服务器端 LLM 自动提取事实（例如，“用户更喜欢 Python”、“用户在 Acme Corp 工作”）——您不必告诉它要记住什么。

### 3. 下一回合的后台预取

在同步的同时，Hermes 开始在 Mem0 上进行后台搜索，为下一回合预加载相关记忆。当您键入下一条消息时，内存已被缓存。## 代理工具

当 Mem0 处于活动状态时，LLM 会获得三个可以在对话期间调用的额外工具：

|工具|描述 |
|------|-------------|
| `mem0_profile` |获取有关用户的所有存储记忆 |
| `mem0_搜索` |通过内存进行语义搜索（支持通过“rerank”和“top_k”参数进行可选的重新排名）|
| `mem0_结论` |逐字存储特定事实 — 使用“infer=False”，因此不会发生服务器端 LLM 提取 |

## 安装

安装 Hermes 代理：```bash
curl -fsSL https://raw.githubusercontent.com/NousResearch/hermes-agent/main/scripts/install.sh | bash
source ~/.bashrc
```
当您启用 Mem0 提供程序时，会自动安装 `mem0ai` Python 包 - 无需手动 pip 安装。

## 设置

### 选项 1：交互式安装向导（推荐）```bash
hermes memory setup
```
选择 **mem0** 作为提供程序，并在出现提示时输入您的 Mem0 API 密钥。向导会将您的配置写入“~/.hermes/mem0.json”。

<Note>从 <a href="https://app.mem0.ai?utm_source=oss&utm_medium=integration-hermes" rel="nofollow">app.mem0.ai</a> 获取 API 密钥。</Note>

### 选项 2：手动配置```bash
hermes config set memory.provider mem0
echo "MEM0_API_KEY=your-api-key" >> ~/.hermes/.env
```
然后在你的`config.yaml`中：```yaml
memory:
  provider: mem0
```
就是这样——Mem0 从此时开始自动运行。

## 配置选项

配置存储在`~/.hermes/mem0.json`中。也可以通过环境变量设置值。

|关键|环境变量 |默认|描述 |
|-----|-------------|---------|-------------|
| `api_key` | `MEM0_API_KEY` | — | **必填。** Mem0 平台 API 密钥 |
| `用户 ID` | `MEM0_USER_ID` | `hermes 用户` |用于确定内存范围的用户标识符 |
| `agent_id` | `MEM0_AGENT_ID` | “爱马仕”|代理标识符|
| `重新排名` | — | `真实` |启用内存重新排序 |


## 可靠性

- **断路器** — 如果 Mem0 的 API 连续失败 5 次，Hermes 将停止调用它 2 分钟，然后重试。在此期间，代理可以正常工作，没有记忆。
- **非阻塞** — 所有 Mem0 API 调用都发生在后台守护线程中。缓慢或失败的 API 调用永远不会阻止您的对话。- **线程安全** — Mem0 客户端使用带有锁定的延迟初始化，对于并发访问是安全的。

## 主要特点1. **零延迟调用** — 内存在后台预取并缓存，在您键入之前准备就绪
2. **服务器端提取** — Mem0 的 API 自动从每个交换中提取事实并删除重复数据
3. **非阻塞** — 所有 API 调用都在后台守护线程中运行
4. **容错** - 即使 Mem0 暂时无法访问，断路器也能确保代理正常工作
