通过集成 ElevenLabs 和 Mem0 创建具有记忆功能的基于语音的对话式 AI 代理。这种集成可以实现持久的、上下文感知的语音交互，并记住过去的对话。

## 概述

在本指南中，我们将构建一个语音代理：
1.使用ElevenLabs Conversational AI进行语音交互
2. 利用 Mem0 存储和检索过去对话的记忆
3. 根据用户历史记录提供个性化响应

## 集成代码分解

让我们将实现分解为可管理的部分：

### 1. 导入和环境设置

首先，我们导入所需的库并设置环境：```python
import os
import signal
import sys
from mem0 import AsyncMemoryClient

from elevenlabs.client import ElevenLabs
from elevenlabs.conversational_ai.conversation import Conversation
from elevenlabs.conversational_ai.default_audio_interface import DefaultAudioInterface
from elevenlabs.conversational_ai.conversation import ClientTools
```
这些进口产品提供：
- 用于系统操作和信号处理的标准Python库
- 来自 Mem0 的“AsyncMemoryClient”，用于内存操作
- 用于语音交互的 ElevenLabs 组件

### 2.环境变量和验证

接下来，我们验证所需的环境变量：```python
def main():
    # Required environment variables
    AGENT_ID = os.environ.get('AGENT_ID')
    USER_ID = os.environ.get('USER_ID')
    API_KEY = os.environ.get('ELEVENLABS_API_KEY')
    MEM0_API_KEY = os.environ.get('MEM0_API_KEY')

    # Validate required environment variables
    if not AGENT_ID:
        sys.stderr.write("AGENT_ID environment variable must be set\n")
        sys.exit(1)

    if not USER_ID:
        sys.stderr.write("USER_ID environment variable must be set\n")
        sys.exit(1)

    if not API_KEY:
        sys.stderr.write("ELEVENLABS_API_KEY not set, assuming the agent is public\n")

    if not MEM0_API_KEY:
        sys.stderr.write("MEM0_API_KEY environment variable must be set\n")
        sys.exit(1)

    # Set up Mem0 API key in the environment
    os.environ['MEM0_API_KEY'] = MEM0_API_KEY
```
本节：
- 检索所需的环境变量
- 执行验证以确保存在所需的变量
- 如果缺少所需的变量，则退出应用程序并显示错误消息
- 设置环境中的Mem0 API密钥供Mem0客户端使用

### 3.客户端初始化

初始化 ElevenLabs 和 Mem0 客户端：```python
    # Initialize ElevenLabs client
    client = ElevenLabs(api_key=API_KEY)

    # Initialize memory client and tools
    client_tools = ClientTools()
    mem0_client = AsyncMemoryClient()
```
在这里我们：
- 使用 API 密钥创建 ElevenLabs 客户端
- 初始化一个ClientTools对象，用于注册功能工具
- 为 Mem0 交互创建 AsyncMemoryClient 实例

### 4. 记忆功能定义

定义将注册为工具的两个关键内存函数：```python
    # Define memory-related functions for the agent
    async def add_memories(parameters):
        """Add a message to the memory store"""
        message = parameters.get("message")
        await mem0_client.add(
            messages=message,
            user_id=USER_ID
        )
        return "Memory added successfully"

    async def retrieve_memories(parameters):
        """Retrieve relevant memories based on the input message"""
        message = parameters.get("message")

        # For Platform API, user_id goes in filters
        filters = {"user_id": USER_ID}

        # Search for relevant memories using the message as a query
        results = await mem0_client.search(
            query=message,
            filters=filters
        )

        # Extract and join the memory texts
        memories = ' '.join([result["memory"] for result in results.get('results', [])])
        print("[ Memories ]", memories)

        if memories:
            return memories
        return "No memories found"
```
这些功能：

#### `添加内存`:
- 采用包含要记住的信息的消息参数
- 使用“add”方法将消息存储在Mem0中
- 将内存与特定的USER_ID关联起来
- 向代理返回成功消息

#### `检索记忆`:
- 将消息参数作为搜索查询
- 设置过滤器仅检索当前用户的记忆
- 使用语义搜索来查找相关记忆
- 将所有检索到的记忆合并成一个文本
- 将检索到的内存打印到控制台以进行调试
- 返回记忆，如果没有找到，则返回“未找到记忆”消息

### 5. 将内存函数注册为工具

使用 ElevenLabs ClientTools 系统注册内存函数：```python
    # Register the memory functions as tools for the agent
    client_tools.register("addMemories", add_memories, is_async=True)
    client_tools.register("retrieveMemories", retrieve_memories, is_async=True)
```
这允许 ElevenLabs 代理：
- 通过函数调用访问这些函数
- 等待异步结果（is_async=True）
- 按名称调用这些函数（“addMemories”和“retrieveMemories”）

### 6. 对话设置

配置与 ElevenLabs 的对话：```python
    # Initialize the conversation
    conversation = Conversation(
        client,
        AGENT_ID,
        # Assume auth is required when API_KEY is set
        requires_auth=bool(API_KEY),
        audio_interface=DefaultAudioInterface(),
        client_tools=client_tools,
        callback_agent_response=lambda response: print(f"Agent: {response}"),
        callback_agent_response_correction=lambda original, corrected: print(f"Agent: {original} -> {corrected}"),
        callback_user_transcript=lambda transcript: print(f"User: {transcript}"),
        # callback_latency_measurement=lambda latency: print(f"Latency: {latency}ms"),
    )
```
这将建立对话：
- ElevenLabs 客户端和代理 ID
- 基于 API 密钥存在的身份验证要求
- 用于处理音频 I/O 的 DefaultAudioInterface
- 具有我们的记忆功能的client_tools
- 回调函数：
  - 显示代理响应
  - 显示更正的答案（当客服人员自我更正时）
  - 显示用户记录以进行调试
  -（注释掉）延迟测量

### 7. 对话管理

开始并管理对话：```python
    # Start the conversation
    print(f"Starting conversation with user_id: {USER_ID}")
    conversation.start_session()

    # Handle Ctrl+C to gracefully end the session
    signal.signal(signal.SIGINT, lambda sig, frame: conversation.end_session())

    # Wait for the conversation to end and get the conversation ID
    conversation_id = conversation.wait_for_session_end()
    print(f"Conversation ID: {conversation_id}")


if __name__ == '__main__':
    main()
```
最后一部分：
- 打印一条消息，指示对话已开始
- 开始对话会话
- 设置信号处理程序以通过 Ctrl+C 优雅地结束会话
- 等待会话结束并获取对话ID
- 打印对话ID以供参考

## 内存工具概述

这种集成为您的对话式 AI 代理提供了两个关键的记忆功能：

### 1. 添加记忆 (`addMemories`)

“addMemories”工具允许您的代理在对话期间存储重要信息，包括：
- 用户偏好
- 用户分享的重要事实
- 谈话期间做出的决定或承诺
- 需要跟进的行动项目

当代理识别出值得记住的信息时，它会调用此函数，以使用适当的用户 ID 将其存储在 Mem0 数据库中。

#### 工作原理：
1.代理识别应该记住的信息
2. 将信息格式化为消息字符串
3. 使用此消息调用 `addMemories` 函数
4. 该函数将内存存储在Mem0中并链接到用户的ID
5. 以后的对话可以找回这段记忆

#### 代理提示中的用法示例：```
When the user shares important information like preferences or personal details, 
use the addMemories function to store this information for future reference.
```
### 2. 检索记忆 (`retrieveMemories`)

“retrieveMemories”工具允许您的代理从之前的对话中搜索并检索相关记忆。代理人可以：
- 搜索与当前主题相关的上下文
- 调用用户偏好
- 记住以前在类似主题上的互动
- 在多个会话中创建连续性

#### 工作原理：
1. 代理需要当前对话的上下文
2.它使用当前对话主题或问题调用“retrieveMemories”
3.该函数在Mem0中执行语义搜索
4.相关记忆返回给代理
5. 代理将这些记忆纳入其响应中

#### 代理提示中的用法示例：```
At the beginning of each conversation turn, use retrieveMemories to check if we've 
discussed this topic before or if the user has shared relevant preferences.
```

You are a helpful voice assistant that remembers past conversations with the user.

  * User preferences
  * Personal details the user shares
  * Important decisions made

## 对话流程示例

以下是与记忆的典型对话的流程：

1. **用户说话**：“嗨，你还记得我最喜欢的颜色吗？”

2. **Agent检索记忆**：   ```python
   # Agent calls retrieve_memories
   memories = retrieve_memories({"message": "user's favorite color"})
   # If found: "The user's favorite color is blue"
   ```
3. **具有上下文的代理进程**：
   - 如果找到记忆：准备个性化的回复
   - 如果没有记忆：准备询问并存储信息

4. **代理回复**：
   - 凭记忆：“是的，你最喜欢的颜色是蓝色！”
   - 没有记忆：“我想你以前没有告诉过我你最喜欢的颜色。它是什么？”

5. **用户响应**：“它实际上是绿色的。”

6. **代理存储新信息**：   ```python
   # Agent calls add_memories
   add_memories({"message": "The user's favorite color is green"})
   ```
7. **特工确认**：“谢谢，我会记住你最喜欢的颜色是绿色。”

## 故障排除

- **缺少 API 密钥**： 
  - 错误：“必须设置 API_KEY 环境变量”
  - 解决方案：确保 .env 文件或系统环境中的所有环境变量设置正确
  
- **连接问题**：
  - 错误：“无法连接到 API”
  - 解决方案：检查您的网络连接和 API 密钥权限。验证 API 密钥是否有效并具有必要的权限。
  
- **清空内存结果**：
  - 症状：特工总是回复“未找到记忆”
  - 解决方案：这对于新用户来说是正常现象。随着对话的发生，记忆数据库会随着时间的推移而建立。您的查询也可能在语义上与存储的记忆不相似 - 尝试不同的措辞。- **特工不使用记忆**：
  - 症状：代理检索记忆但未将其合并到响应中
  - 解决方案：更新代理的提示，明确指示它在响应中使用检索到的记忆
