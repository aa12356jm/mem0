## 选择正确的重排序器- **API 优先** 当您需要最高质量并且可以吸收请求成本时（Cohere、零熵）。  
- **自托管**，用于必须保留在您的硬件上的隐私敏感部署（Sentence Transformer、Hugging Face）。  
- **LLM驱动**当您需要定制评分逻辑或复杂提示时。  
- **混合**，仅在高级旅程上启用重新排名以控制支出。

## 实施清单

1. 确认基线搜索 KPI，以便您可以衡量提升情况。  
2. 选择一个提供商并将“reranker”块添加到您的配置中。  
3. 使用类似生产的查询批次测试延迟影响。  
4. 通过“rerank”标志决定是否启用全局重新排名或每次搜索启用重新排名。
