[Supabase](https://supabase.com/) 是一个开源 Firebase 替代方案，它提供带有 pgvector 扩展的 PostgreSQL 数据库，用于矢量相似性搜索。它为存储和查询向量嵌入提供了强大且可扩展的解决方案。

创建 [Supabase](https://supabase.com/dashboard/projects) 帐户和项目，然后从项目设置 > 数据库获取连接字符串。有关详细信息，请参阅[文档](https://supabase.github.io/vecs/hosting/)。

### TypeScript 实现的 SQL 迁移

需要以下 SQL 迁移来启用向量扩展并创建内存表：```sql
-- Enable the vector extension
create extension if not exists vector;

-- Create the memories table
create table if not exists memories (
  id text primary key,
  embedding vector(1536),
  metadata jsonb,
  created_at timestamp with time zone default timezone('utc', now()),
  updated_at timestamp with time zone default timezone('utc', now())
);

-- Create the vector similarity search function
create or replace function match_vectors(
  query_embedding vector(1536),
  match_count int,
  filter jsonb default '{}'::jsonb
)
returns table (
  id text,
  similarity float,
  metadata jsonb
)
language plpgsql
as $$
begin
  return query
  select
    t.id::text,
    1 - (t.embedding <=> query_embedding) as similarity,
    t.metadata
  from memories t
  where case
    when filter::text = '{}'::text then true
    else t.metadata @> filter
  end
  order by t.embedding <=> query_embedding
  limit match_count;
end;
$$;
```
转到 [Supabase](https://supabase.com/dashboard/projects) 并在 SQL 编辑器中运行上述 SQL 迁移。### 配置

### 索引方法

支持以下索引方法：- `auto`：自动选择最佳可用索引方法
- `hnsw`：分层可导航小世界图索引（更快的搜索，更多的内存使用）
- `ivfflat`：倒排文件平面索引（速度和内存的良好平衡）

### 距离测量

用于相似性搜索的可用距离度量：

- `cosine_distance`：余弦相似度（推荐用于大多数嵌入模型）
- `l2_distance`：欧氏距离
- `l1_distance`：曼哈顿距离
- `max_inner_product`：最大内积相似度

### 最佳实践

1. **索引方法选择**：
   - 当内存不受限制时，使用“hnsw”可获得最快的搜索性能
   - 使用“ivfflat”在搜索速度和内存使用之间取得良好的平衡
   - 如果不确定，请使用“auto”，它将根据您的数据选择最佳方法

2.**距离测量选择**：
   - 对大多数嵌入模型（OpenAI、Hugging Face 等）使用 `cosine_distance`
   - 如果您的向量已标准化，请使用“max_inner_product”
   - 如果使用原始特征向量，请使用“l2_distance”或“l1_distance”3. **连接字符串**：
   - 始终对连接字符串中的敏感信息使用环境变量
   - 格式：`postgresql://用户:密码@主机:端口/数据库`
