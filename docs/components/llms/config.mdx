## 如何定义配置？

<标签>
  <标签标题=“Python”>
    `config` 被定义为一个具有两个主键的 Python 字典：
    - `llm`：指定 llm 提供程序及其配置
      - `provider`：llm 的名称（例如“openai”、“groq”）
      - `config`：包含特定于提供者的设置的嵌套字典
  </标签>
  <选项卡标题=“TypeScript”>
    `config` 被定义为具有以下键的 TypeScript 对象：
    - `llm`：指定LLM提供者及其配置（必需）
      - `provider`：法学硕士的名称（例如、“openai”、“groq”）
      - `config`：包含特定于提供者的设置的嵌套对象
    - `embedder`：指定嵌入器提供程序及其配置（可选）
    - `vectorStore`：指定矢量存储提供程序及其配置（可选）
    - `historyDbPath`：历史数据库文件的路径（可选）
  </标签>
</标签>

### 配置值优先级

配置值按以下优先顺序应用（从最高到最低）：1. 在`config`对象/字典中显式设置的值
2. 环境变量（例如`OPENAI_API_KEY`、`OPENAI_BASE_URL`）
3. LLM实现中定义的默认值

这意味着“config”中指定的值将覆盖相应的环境变量，进而覆盖默认值。

## 如何使用配置

以下是如何将配置与 Mem0 一起使用的一般示例：

<代码组>```python Python
import os
from mem0 import Memory

os.environ["OPENAI_API_KEY"] = "sk-xx" # for embedder

config = {
    "llm": {
        "provider": "your_chosen_provider",
        "config": {
            # Provider-specific settings go here
        }
    }
}

m = Memory.from_config(config)
m.add("Your text here", user_id="user", metadata={"category": "example"})

```

```typescript TypeScript
import { Memory } from 'mem0ai/oss';

// Minimal configuration with just the LLM settings
const config = {
  llm: {
    provider: 'your_chosen_provider',
    config: {
      // Provider-specific settings go here
    }
  }
};

const memory = new Memory(config);
await memory.add("Your text here", { userId: "user123", metadata: { category: "example" } });
```
</代码组>

## 为什么需要配置？

配置对于以下方面至关重要：
1. 指定使用哪个法学硕士。
2. 提供必要的连接详细信息（例如型号、api_key、温度）。
3. 确保正确初始化并连接到您选择的法学硕士。
