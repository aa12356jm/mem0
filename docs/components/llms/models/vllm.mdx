[vLLM](https://docs.vllm.ai/) 是一种适用于大型语言模型的高性能推理引擎，可为本地推理提供显着的性能改进。它旨在最大限度地提高服务法学硕士的吞吐量和内存效率。

## 先决条件

1. **安装vLLM**：   ```bash
   pip install vllm
   ```
2. **启动 vLLM 服务器**：   ```bash
   # For testing with a small model
   vllm serve microsoft/DialoGPT-medium --port 8000

   # For production with a larger model (requires GPU)
   vllm serve Qwen/Qwen2.5-32B-Instruct --port 8000
   ```
＃＃ 用法```python
import os
from mem0 import Memory

os.environ["OPENAI_API_KEY"] = "your-api-key"  # used for embedding model

config = {
    "llm": {
        "provider": "vllm",
        "config": {
            "model": "Qwen/Qwen2.5-32B-Instruct",
            "vllm_base_url": "http://localhost:8000/v1",
            "temperature": 0.1,
            "max_tokens": 2000,
        }
    }
}

m = Memory.from_config(config)
messages = [
    {"role": "user", "content": "I'm planning to watch a movie tonight. Any recommendations?"},
    {"role": "assistant", "content": "How about thriller movies? They can be quite engaging."},
    {"role": "user", "content": "I'm not a big fan of thrillers, but I love sci-fi movies."},
    {"role": "assistant", "content": "Got it! I'll avoid thrillers and suggest sci-fi movies instead."}
]
m.add(messages, user_id="alice", metadata={"category": "movies"})
```
## 配置参数

|参数|描述 |默认 |环境变量 |
| ---------------- | --------------------------------- | -------------------------------------- | -------------------- |
| `模型` |在 vLLM 服务器上运行的模型名称 | `"Qwen/Qwen2.5-32B-指令"` | - |
| `vllm_base_url` | vLLM 服务器 URL | `"http://localhost:8000/v1"` | `VLLM_BASE_URL` |
| `api_key` | API 密钥（本地虚拟）| `"vllm-api-key"` | `VLLM_API_KEY` |
| `温度` |取样温度| `0。1`| - |
| `max_tokens` |生成的最大代币 | `2000` | - |

## 环境变量

您可以设置这些环境变量，而不是在配置中指定它们：```bash
export VLLM_BASE_URL="http://localhost:8000/v1"
export VLLM_API_KEY="your-vllm-api-key"
export OPENAI_API_KEY="your-openai-api-key"  # for embeddings
```
## 好处

- **高性能**：推理速度比标准实现快 2-24 倍
- **内存高效**：通过 PagedAttention 优化内存使用
- **本地部署**：保持数据私密性并降低 API 成本
- **轻松集成**：直接替代其他 LLM 提供商
- **灵活**：适用于 vLLM 支持的任何模型

## 故障排除

1. **服务器没有响应**：确保 vLLM 服务器正在运行   ```bash
   curl http://localhost:8000/health
   ```
2. **404错误**：确保基本URL格式正确   ```python
   "vllm_base_url": "http://localhost:8000/v1"  # Note the /v1
   ```
3. **未找到模型**：检查模型名称是否与服务器匹配

4. **内存不足**：尝试较小的模型或减少“max_model_len”   ```bash
   vllm serve Qwen/Qwen2.5-32B-Instruct --max-model-len 4096
   ```
